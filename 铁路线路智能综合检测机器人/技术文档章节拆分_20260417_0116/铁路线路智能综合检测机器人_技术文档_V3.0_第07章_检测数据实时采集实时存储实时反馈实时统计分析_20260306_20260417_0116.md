## 07 检测数据实时采集、实时存储、实时反馈、实时统计分析

### 一、实时采集

| 数据类型 | 来源 | 采集频率 | 说明 |
|---|---|---|---|
| 2D 图像流 | 6 路工业相机 | 20~25 fps/路 | 工业协议，实时帧同步 |
| 3D 点云流 | 2 路 3D 线激光 | 实时 | 轮廓数据，垂直朝下扫描 |
| 姿态参数流 | 姿态传感器HWT905（1路） | 毫秒级 | 水平/高低角度值 |
| 里程数据 | 轮速传感器 | 实时 | 帧编号 + 里程绑定 |
| 设备状态流 | 各传感器节点 | 秒级 | 温度/电压/帧率等 |

### 二、实时存储

| 存储位置 | 介质 | 说明 |
|---|---|---|
| 本地存储（工控机） | SSD | 检测过程中实时写入，断网不丢数 |
| 云端存储（远程） | 云服务器 | 移动网络实时同步上传，双重备份 |

**存储数据内容：**

- 原始图像流（带帧编号 + 里程 + 时间戳）
- 3D 点云数据（带同步标签）
- 姿态参数数据（带时间戳）
- AI 判定结果（缺陷类型 + 等级 + 位置）
- 设备状态日志（传感器健康 / 用电状态）

### 三、实时反馈

| 反馈内容 | 触发条件 | 反馈方式 |
|---|---|---|
| 缺陷报警 | AI 识别到 ≥一级缺陷 | 本地声光 + 云端推送，秒级响应 |
| 任务进度 | 实时 | 触摸屏 + 云端实时显示里程/帧数/检测量 |
| 设备异常 | 传感器离线/帧率异常/电压过限 | 云端告警，远程可查 |
| 数据完整性报告 | 任务结束后 | 自动生成数据完整性报告 |

> **大数据是平台的根基**：每一次检测任务产生的数据，实时汇入云端大数据平台，为健康趋势分析、养护决策提供数据资产。

### 四、数据的实时分析统计

本系统不仅采集和存储数据，更对采集到的数据进行实时分析统计，让数据在检测过程中即刻产生价值。

**实时统计内容：**

| 统计项目 | 数据来源 | 更新频率 | 说明 |
|---|---|---|---|
| 已采集里程 | 轮速传感器 | 实时 | 当前任务已检测里程，精确到米 |
| 采集帧数统计 | 6路相机 | 实时 | 各相机已采集帧数，丢包率统计 |
| 缺陷数量统计 | AI 推理结果 | 实时 | 各类型缺陷（裂纹/掉块/松动等）实时计数 |
| 缺陷密度分布 | AI 推理 + 里程 | 实时 | 按里程区段统计缺陷分布密度 |
| 任务进度 | 综合计算 | 实时 | 当前任务完成百分比，预计剩余时间 |
| 数据完整性 | 存储模块 | 实时 | 本地写入量 / 应写入量，丢帧率统计 |

**实时分析的价值：**

- **任务中期预警**：检测过程中若缺陷数量突增，系统立即告警，提醒重点关注该区段
- **数据质量监控**：实时丢包率显示，确保数据采集有效完整
- **任务进度透明**：操作人员和监控中心随时掌握任务完成情况
- **动态调整依据**：根据实时缺陷密度，运维人员可动态调整后续养护优先级

> **实时分析是检测任务的仪表盘**：让检测过程从黑箱作业变为全程透明可控，数据在产生的瞬间就被分析、被利用。

### 六、存储系统详细架构

**本地存储设计：**

| 项目 | 规格 | 说明 |
|---|---|---|
| 存储介质 | 2TB NVMe SSD | 数据盘，RAID1冗余 |
| 写入速度 | ≥ 3.5 GB/s 顺序写入 | 满足6路相机+2路激光实时写入 |

| 本地保留策略 | 最近10次任务 | 自动循环覆盖最旧数据 |

### 七、数据传输系统

**传输策略：**

| 场景 | 传输方式 | 说明 |
|---|---|---|
| 作业中实时回传 | 4G/5G 无线 | 有缺陷帧优先上传，非缺陷帧本地缓存 |
| 任务结束后批量上传 | WiFi/有线 | 任务结束后在停车场/工区高速回传 |
| 断网续传 | 本地缓存 | 网络恢复后自动从断点继续，不重传 |
| 大文件优先策略 | 点云优先 | 点云数据大于图像，优先占带宽 |

**4G/5G带宽需求分析：**

| 项目 | 数值 | 说明 |
|---|---|---|
| 缺陷帧优先上传 | 每帧 ~1.5MB | 检出缺陷即上传，延迟 < 5s |
| 普通帧本地缓存 | ~150 MB/公里 | 非缺陷数据暂存本地 |
| 上传带宽需求 | ≥ 10 Mbps | 满足缺陷帧实时回传 |
| 任务结束后批量上传 | 100 Mbps+ | WiFi/有线环境下高速回传剩余数据 |

**断点续传机制：**

```
任务中网络中断 → 数据本地缓存（含断点标记）
                                    ↓
网络恢复 → 自动从断点继续上传 → 云端校验已接收数据
                                    ↓
                    已传数据跳过 → 只传剩余未传数据
```

### 九、数据完整性保障

| 保障措施 | 说明 |
|---|---|
| 写入校验 | 每帧写入后读回验证，损坏帧自动重写 |
| 任务清单（manifest） | 任务开始生成清单，结束后逐一核对，丢帧位置标记 |
| 完整性报告 | 任务结束自动生成数据完整性报告，含：总帧数/丢帧数/损坏帧数/完整性百分比 |
| RAID1 数据冗余 | SSD 支持 RAID1，任意一块盘损坏数据不丢失 |
| 云端校验 | 数据上传后云端 SHA-256 对比，不一致自动重传 |
| 备份策略 | 本地保留10次任务，云端保留全部任务数据 |

### 十、实时统计详细算法

| 统计指标 | 计算方式 | 单位 |
|---|---|---|
| 已采集里程 | 轮速计数器累加，精度 ±1mm | 米（km） |
| 采集帧数 | 各相机帧计数器，分别统计 | 帧 |
| 丢帧率 | 应收帧数 / 实收帧数 × 100% | % |
| 缺陷数量 | AI结果中各类型缺陷计数 | 个 |
| 缺陷密度 | 缺陷数量 / 已采集里程 | 个/km |
| 任务进度 | 已采集里程 / 设定里程 × 100% | % |
| 数据完整性 | 实收帧数 / 应收帧数 × 100% | % |

**丢帧补偿算法：**

- 期望帧数 = 里程 / 每帧覆盖距离
- 应收帧数 = 期望帧数 × 帧率校正系数
- 实收帧数 = 实际写入帧数（扣除损坏帧）
- 丢帧率 = (应收 - 实收) / 应收 × 100%
