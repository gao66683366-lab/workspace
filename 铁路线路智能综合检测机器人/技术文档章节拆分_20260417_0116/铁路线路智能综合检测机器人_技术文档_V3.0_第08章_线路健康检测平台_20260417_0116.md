## 08 线路健康检测平台

### 一、大数据平台定位

大数据平台是本系统的核心资产平台：

> **检测数据 → 实时汇聚 → 积累沉淀 → 挖掘分析 → 决策支撑**

没有大数据，就没有线路健康分析、没有趋势预测、没有科学的养护决策。

### 二、平台数据来源

| 数据来源 | 数据内容 | 数据量 |
|---|---|---|
| 每次检测任务 | 图像流 + 点云流 + 姿态流 + 缺陷判定结果 | ~TB级/公里 |
| 设备运行日志 | 传感器状态 / 用电数据 / 通信日志 | GB级/月 |
| 养护记录 | 养护时间 / 内容 / 效果验证数据 | MB级/次 |
| 环境数据 | 温度 / 湿度 / 气候（可选接入） | 关联分析用 |

### 三、平台核心功能

**数据汇聚与存储**

- 多任务数据自动汇聚，按里程区段组织
- 时空索引建立，支持按里程/时间/病害类型多维检索
- 历史数据可回放，每帧数据可追溯

**健康热力图**

- 按里程区段绿/黄/红三色，直观显示全线健康状况
- 可切换查看不同病害类型的分布

**趋势分析**

- 同一区段历史检测数据对比，识别劣化趋势
- 预测未来 1/3/6 个月病害数量发展

**健康综合评分（0~100）**

| 评分区间 | 健康等级 | 养护建议 |
|---|---|---|
| 90~100 | 优秀 | 正常巡检，维持现状 |
| 75~89 | 良好 | 关注重点区段，计划性养护 |
| 60~74 | 一般 | 加强监测，优先安排养护 |
| 40~59 | 较差 | 尽快安排养护，缩短巡检周期 |
| 0~39 | 危险 | 立即处置，启动应急养护 |

**报表自动生成**

- 按需自动生成检测报告，支持导出 Word/PDF
- 包含缺陷清单 + 健康评分 + 趋势分析 + 养护建议

## 09 智能养护决策系统

### 一、智能养护决策逻辑

系统以 AI 为驱动，基于大数据分析结果，自动生成科学养护决策：

```
检测数据 ──→ 大数据分析 ──→ 养护优先级排序 ──→ 养护计划生成
                  │                                    │
                  ↓                                    ↓
            健康趋势预测           养护效果验证
                  │                                    │
                  ↓                                    ↓
           预测性维护建议         二次检测数据闭环
```

### 二、养护优先级算法

**基础优先级规则：**

| 病害等级 | 优先级 |
|---|---|
| 三级缺陷 | P0 — 立即处理 |
| 二级缺陷 | P1 — 纳入计划养护 |
| 一级缺陷 | P2 — 重点跟踪观察 |

**同等级优先排序规则：**
- 趋势恶化区段 > 稳定区段
- 历史高发区段 > 偶发区段
- 重点线路（重载/弯道/桥隧）优先

**输出：** 自动生成"本周养护建议清单"，含：里程 + 位置 + 病害类型 + 等级 + 建议措施

### 三、AI 全说明决策支撑

每项养护建议均附带完整依据：

- 病害定位：里程区段 + 精确位置
- 判定依据：AI 识别结果 + 偏差数值 + 历史对比趋势
- 养护建议：对应的标准依据 + 推荐措施
- 预估工时：基于病害类型和范围估算

> 用户可知晓"AI 为什么要建议现在修这里而不是别处"，决策过程全透明。

### 四、养护效果量化验证

```
养护执行 ──→ 二次检测 ──→ 数据对比 ──→ 效果量化评分
                                      │
                          效果达标 ←── └──→ 效果不达标→重新制定方案
```

## 10 设备完好性保障与远程监控

### 一、设备完好性保障体系

在检测过程中，对机器人上所有检测设备进行实时完好性监控，确保设备始终处于良好工作状态。设备安全稳定运行是检测数据质量的根基。

```
┌─────────────────────────────────────────────────┐
│           检测设备完好性实时监控                  │
│                                                  │
│  6路工业相机 ──→ 在线状态 / 帧率 / 丢包率 / 温度 │
│  2路3D激光 ──→ 功率 / 数据率 / 温度             │
│  2路陀螺仪 ──→ 角速度 / 姿态角 / 健康状态        │
│  工控机本体 ──→ CPU / 内存 / 磁盘 / 进程 / 网络  │
│  工业总线 ────→ 通信状态 / 故障码               │
│  通信链路 ────→ 移动网络信号 / 带宽 / 延迟       │
└─────────────────────────────────────────────────┘
```

**完好性监控指标：**

| 设备 | 监控参数 | 告警阈值 |
|---|---|---|
| 工业相机 ×6 | 帧率 / 丢包率 / 在线状态 / 温度 | 帧率 <20fps / 丢包率>1% / 离线 |
| 3D 激光 ×2 | 数据率 / 温度 / 功率 | 数据率异常 / 温度超限 |
| 双陀螺仪 | 角速度漂移 / 姿态角异常 | 漂移超限 |
| 工控机 | CPU占用>80% / 内存>85% / 磁盘<10GB | 超限告警 |
| 工业总线 | 通信状态 / 故障码 | 通信中断 / 故障码非零 |
| 移动网络链路 | 信号强度 / 带宽 / 延迟 | 信号<-100dBm / 延迟>500ms |

### 二、智能安全供电管理

用电安全管理是保障机器人安全运行的核心，贯穿检测任务全流程。

本系统的智能安全供电管理体系覆盖所有用电节点，实现从蓄电池到各传感器设备的全链路电流、电压、温度实时监控，确保设备在检测过程中不会因电力问题而受损或停机。

**蓄电池安全管理 — 全参数实时监控**

蓄电池是检测机器人的"心脏"，本系统对蓄电池实施全参数实时监控：

| 监控参数 | 说明 | 安全机制 |
|---|---|---|
| **电压** | 实时监控蓄电池组电压 | 低于阈值立即报警并触发返航，防止过放 |
| **电流** | 充放电电流实时监测 | 充电过流自动断充，放电异常预警 |
| **SOC 电量** | 剩余电量百分比实时显示 | 电量低于预设阈值前主动报警 |
| **SOH 健康度** | 电池健康状态评估 | 健康度低于阈值时提示更换 |
| **续航预估** | 基于当前电量 + 历史能耗 | 实时预估剩余可行驶里程，提前规划返航 |
| **BMS 电路板温度** | 蓄电池管理系统电路板温度实时采集 | 电路板温度超限立即报警，防止BMS失效引发安全事故 |
| **智能电路管理板温度** | 智能电源管理板温度实时采集 | 管理板温度超限自动降载或断电，防止过热损坏 |

**三级保护机制：**

| 级别 | 触发条件 | 动作 |
|---|---|---|
| 预警 | 电量 <20% 或 电压接近低限 | 云端推送预警，提醒准备返航 |
| 告警 | 电量 <10% 或 温度超限 | 声光报警 + 强制触发一键返航 |
| 保护 | 电压/电流严重超限 | 硬件独立保护回路立即切断供电，保护设备和线路 |

> BMS 电路板和智能电路管理板的温度实时采集，是防止蓄电池管理系统自身失效引发安全事故的最后一道防线，填补了传统监控方案的关键盲区。

**检测设备用电管理 — 全节点电流电压实时监控**

整机所有用电节点均纳入实时监控，任何电流电压异常均可快速定位：

| 用电节点 | 监控参数 | 说明 |
|---|---|---|
| 6路工业相机 | 电压 / 电流 / 功率 | 相机节点用电状态实时监控 |
| 2路3D激光 | 电压 / 电流 / 功率 | 激光器节点用电状态实时监控 |
| 2路双陀螺仪 | 电压 / 电流 | 陀螺仪节点用电状态实时监控 |
| 工控机 | 电压输入 / 电流 / 功率 | 主控节点用电实时监控 |
| 工业总线伺服驱动 | 电压 / 电流 / 故障码 | 运动控制节点用电和安全监控 |

> **实时反馈机制**：所有设备的电流、电压数据**每秒刷新**，异常节点可在云端**秒级定位**，大幅缩短故障排查时间。

**运动控制安全保障 — 工业总线硬实时守护**

| 监控项 | 说明 |
|---|---|
| 工业总线通信状态 | 实时监控通信是否中断 |
| 伺服驱动状态 | 实时获取驱动故障码 |
| 急停回路状态 | 硬件独立急停回路，失效导向安全 |
| 运行速度一致性 | 定速巡航保障采集数据速度稳定 |

### 三、远程监测控制能力

| 能力 | 说明 |
|---|---|
| 实时状态查看 | 云端实时查看任一设备节点的工作状态和用电数据 |
| 参数远程调优 | 远程调整相机曝光/帧率/检测阈值，无需到场 |
| 设备远程重启 | 单节点故障可远程重启恢复，减少现场出勤 |

| 远程固件升级 | 传感器固件可通过移动网络远程推送升级 |

### 四、四类用户价值

| 用户角色 | 使用场景 | 核心价值 |
|---|---|---|

## 11 系统部署与运维保障

### 一、5 步现场部署

| 步骤 | 操作 | 说明 |
|---|---|---|
| 1. 设备就位 | 将检测机器人放置于铁轨 | 4 轮自动对准轨道 |
| 2. 开机自检 | 按下电源开关，系统自动启动自检 | 传感器 / 相机 / 激光自动标定 |

| 5. 任务结束 | 自动返航或手动一键返航 | 数据自动打包上传 |

### 二、运维保障体系

| 保障类型 | 内容 |
|---|---|
| 远程运维 | 移动网络远程接入，故障诊断 + 参数调优 + 固件升级 |
| 定期标定 | 传感器标定（相机内外参 / 激光标定 / 陀螺仪校准） |

| 软件维护 | AI 模型持续迭代，远程推送更新 |
| 数据维护 | 云端数据定期备份，多副本存储 |

### 三、故障恢复机制

| 故障类型 | 处理机制 |
|---|---|
| 单相机离线 | 任务不中断，其他相机覆盖，事后标记数据缺失区段 |
| 单激光离线 | 保留 2D 检测功能，标记降级运行，及时维修 |
| 网络中断 | 本地 SSD 持续存储，网络恢复后自动续传 |
| 工控机死机 | 硬件看门狗自动重启，任务从断点续跑 |
| 电量耗尽 | 自动触发定点返航，不丢已采集数据 |
| 工业总线通信中断 | 硬件急停立即触发，机器人停止，保护设备和线路安全 |
