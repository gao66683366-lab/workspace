# WORKSPACE_MANIFEST

## 目录用途

- `00-agent/`：身份、规则、长期记忆与心跳配置
- `01-project/`：项目结构性资料，按架构/硬件/软件/算法/传输/验收分类
- `02-docx/`：Word 文档，分为草稿、定稿、归档
- `03-scripts/`：脚本工具，按 Python / PowerShell 分类
- `04-analysis/`：分析过程资料与研究性输出
- `05-temp/`：临时文件、中间产物、待清理文本
- `06-skills/`：本地技能与扩展资料
- `99-archive/`：低频历史归档资料

## 文件命名规则

统一采用：`主题_版本号_YYYY-MM-DD.扩展名`

示例：
- `硬件系统架构设计文档_V1.0_2026-03-08.docx`
- `CAM1-CAM6帧率同步分析报告_V1.0_2026-03-08.docx`

## 文档状态规则

- 草稿：文件放入 `02-docx/drafts/`
- 定稿：文件放入 `02-docx/final/`
- 历史废弃版：文件放入 `02-docx/archive/`

## 维护原则

1. 根目录尽量只保留总入口与高频说明文件。
2. 临时文件不得长期停留在根目录。
3. 所有定稿文档统一使用 `.docx`。
4. 输出内容必须严格遵守既定项目口径，不擅自改动已确认事实。
