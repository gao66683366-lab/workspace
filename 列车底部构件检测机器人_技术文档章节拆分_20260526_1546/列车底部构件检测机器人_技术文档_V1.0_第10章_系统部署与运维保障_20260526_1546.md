# 第10章 系统部署与运维保障

## 10.1 现场部署流程

本系统交付后，按以下5步完成现场部署：

### 第一步：环境勘测与方案确认

| 内容 | 说明 |
|---|---|
| 轨道条件勘测 | 轨距、轨型、曲线半径、地面平整度测量 |
| 作业空间评估 | 确认车底净空高度、股道间距、检修台位布局 |
| 网络环境确认 | 确认5G信号、光纤接入点、WiFi覆盖情况 |
| 车型模板规划 | 根据车库内车型，配置对应检测路径模板 |

### 第二步：设备安装与接线

| 内容 | 说明 |
|---|---|
| 机器人展开 | 从运输箱取出，展开轨底平台、装上天线 |
| 有线连接 | 接光纤复合缆至控制室，接手持终端 |
| 固定调试 | 置于测试股道，调整水平位置 |
| 上电开机 | 依次上电，检查各模块启动状态 |

### 第三步：系统标定与测试

| 内容 | 说明 |
|---|---|
| 传感器标定 | IMU零偏标定、里程计校准、倾角零点校准 |
| 相机标定 | 镜头对焦、曝光参数、白平衡、云台角度归零 |
| 激光雷达标定 | 坐标系标定、障碍物检测区域设定 |
| 通信测试 | 有线/5G/WiFi三路通信测试 |
| 运动测试 | 直线行走、转向、越障、爬坡测试 |

### 第四步：试运行与验收

| 内容 | 说明 |
|---|---|
| 单车试检测 | 选取1节车厢，执行完整检测流程 |
| AI功能验证 | 在测试图像上验证AI识别效果 |
| 报告生成验证 | 核查报告内容完整性、准确性 |
| 操控培训 | 现场培训操作人员，考核合格后上岗 |

### 第五步：正式交付与文档移交

| 内容 | 说明 |
|---|---|
| 正式验收 | 按验收标准逐项测试，签署验收报告 |
| 文档移交 | 交付操作手册、维护手册、故障代码表 |
| 备件移交 | 交接备用电池、相机、线缆等备件 |
| 正式上线 | 系统投入正式检修生产 |

## 10.2 运维保障体系

### 10.2.1 日常维护项目

| 维护项 | 周期 | 说明 |
|---|---|---|
| 外观检查 | 每次作业前 | 外观、线缆、连接器 |
| 电池电量 | 每次作业前 | 确认>=80% |
| 镜头清洁 | 每次作业后 | 清洁镜头油污、水渍 |
| 线缆收存 | 每次作业后 | 检查线缆收放机构 |
| 云台归零 | 每次作业后 | 归位至标准位置 |
| 滤光片清洁 | 每周 | 清洁偏振滤光片 |
| 被动均衡充电 | 每月 | 夜间均衡充电 |
| 固件升级 | 按需 | OTA推送最新版本 |

### 10.2.2 定期维护项目

| 维护项 | 周期 | 说明 |
|---|---|---|
| 轨底平台清洁 | 每周 | 清洁轮组、导向机构 |
| 云台润滑 | 每季度 | 检查润滑脂 |
| 电池容量检测 | 每半年 | 深度放电测试，评估SOH |
| 传感器精度校准 | 每半年 | IMU/里程计/倾角复校 |
| 设备密封性检查 | 每半年 | 检查IP67密封 |
| 螺栓紧固检查 | 每季度 | 检查各固定螺栓 |

## 10.3 故障恢复机制

### 10.3.1 故障分级处理

| 级别 | 定义 | 响应要求 | 处理方式 |
|---|---|---|---|
| P1 | 设备无法使用，影响生产 | <=2小时响应 | 现场支持/备机替换 |
| P2 | 核心功能降级，可凑合使用 | <=8小时响应 | 远程诊断+现场支持 |
| P3 | 非核心功能异常 | <=48小时响应 | 远程诊断+计划维护 |

### 10.3.2 备机替换机制

| 场景 | 说明 |
|---|---|
| 现场备机 | 长期使用用户建议配置1台备机 |
| 备机替换流程 | 故障机=>拆下模块=>装上备件=>配置同步=>验证功能 |
| 模块级替换 | 按模块替换（相机/电池/线缆等），无需整机替换 |
| 配置恢复 | 备机接入后，自动从云端同步配置和模型 |

### 10.3.3 数据恢复

| 场景 | 恢复方式 |
|---|---|
| 本地SSD损坏 | 数据从云端备份恢复（关键数据已同步） |
| 任务中断 | 断点续采，从断点位置继续 |
| 系统崩溃 | 重新刷机，配置从配置文件恢复，模型从OTA恢复 |
