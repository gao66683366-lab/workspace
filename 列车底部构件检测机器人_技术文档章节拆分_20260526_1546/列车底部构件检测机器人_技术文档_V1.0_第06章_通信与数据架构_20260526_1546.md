# 第6章 通信与数据架构

## 6.1 通信架构总体设计

本系统通信架构采用**三网物理隔离**设计，将控制网络、采集网络、传输网络完全独立部署，避免各类数据流相互干扰，满足工业控制实时性与数据安全要求。

```
┌──────────────────────────────────────────────────────────────┐
│                   三网物理隔离架构                           │
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │  控制网络     │  │  采集网络     │  │   传输网络        │  │
│  │  EtherCAT    │  │  千兆以太网   │  │  5G/WiFi6/光纤   │  │
│  │  硬实时总线   │  │  相机数据     │  │   远程传输        │  │
│  │  运动驱动    │  │  激光雷达    │  │   远程监控        │  │
│  │  <=1ms响应  │  │  <=200ms延迟 │  │   <=500ms端到端  │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

## 6.2 三网详细说明

### 6.2.1 控制网络（EtherCAT 硬实时总线）

| 项目 | 参数 |
|---|---|
| 协议 | EtherCAT（以太网CAT） |
| 实时性 | 硬实时，响应<=1ms |
| 用途 | 步进驱动控制、电机启停、运动速度环/位置环控制 |
| 拓扑 | 总线式，所有驱动节点串联 |
| 冗余 | 主从站双网口冗余 |
| 安全 | 故障节点自动旁路 |

### 6.2.2 采集网络（千兆以太网）

| 项目 | 参数 |
|---|---|
| 协议 | GigE Vision（工业相机标准） |
| 带宽 | 千兆（>=1Gbps） |
| 用途 | 工业相机视频数据采集、激光雷达点云数据传输 |
| 相机数量 | 支持>=4路4K相机同时接入 |
| 带宽预算 | 4K@30fps×H.265编码≈40Mbps/路，4路≈160Mbps |
| 延迟 | <=200ms（端到端视频延迟） |

### 6.2.3 传输网络（5G/WiFi6/光纤）

| 项目 | 参数 |
|---|---|
| 光纤 | 千兆光纤复合缆，专线直连，延迟<=10ms |
| 5G | 5G NR，室外/远程操控，延迟<=50ms（空口） |
| WiFi 6 | 802.11ax，>=1.2Gbps，室内/库内无线链路 |
| 用途 | 远程监控画面回传、检测数据上传、远程操控指令下发 |
| 切换策略 | 有线优先，自动切换至5G备份 |

## 6.3 通信接口配置

### 6.3.1 主控计算机通信接口

| 接口类型 | 数量 | 用途 |
|---|---|---|
| GigE | 4 | 相机数据采集 |
| EtherCAT | 1 | 运动控制总线 |
| RS485 | 2 | 传感器数据采集（IMU、倾角等） |
| CAN | 1 | 备选传感器通信 |
| USB3.0 | 2 | 外设扩展 |
| HDMI | 1 | 视频输出（本地显示） |
| 光纤接口 | 1 | 有线传输网络 |
| 5G模块 | 1 | 无线传输网络 |
| WiFi模块 | 1 | WiFi 6无线传输 |

### 6.3.2 主控计算机AI推理接口

| 接口 | 说明 |
|---|---|
| GigE Vision | 接收工业相机原始视频帧 |
| OpenVINO | Intel iGPU推理框架接入 |
| SSD | 本地存储检测数据与AI模型 |

### 6.4 数据流分析

### 6.4.1 实时数据流（采集网络）

```
[工业相机] → [GigE接口] → [主控计算机] → [H.265编码] → [本地存储]
                                       → [AI推理（iGPU）] → [缺陷结果]
                                       → [实时视频流] → [手持终端]
```

### 6.4.2 控制数据流（控制网络）

```
[手持终端] → [主控计算机] → [EtherCAT主站] → [步进驱动] → [电机]
```

### 6.4.3 传输数据流（传输网络）

```
[主控计算机] → [5G/光纤] → [远程监控中心]
                              ↕
                        [检修管理系统/MRO平台]
```

## 6.5 数据带宽预算

| 数据类型 | 码率/带宽 | 说明 |
|---|---|---|
| 4K视频流（H.265） | ~40 Mbps/路 | 4路合计~160 Mbps |
| 激光雷达点云 | ~5 Mbps | 16线，10Hz扫描 |
| 控制指令 | <1 Mbps | EtherCAT |
| AI推理结果 | <2 Mbps | 缺陷标记数据 |
| 传感器数据 | <1 Mbps | IMU/里程计/倾角 |
| **峰值总带宽** | **~170 Mbps** | 千兆以太网足够承载 |
| **回传至云端（5G）** | **~50 Mbps** | 经压缩后的关键数据 |

## 6.6 网络安全设计

| 安全措施 | 说明 |
|---|---|
| 三网物理隔离 | 控制网/采集网/传输网完全独立 |
| 数据加密 | TLS 1.3加密传输 |
| 设备认证 | 双向证书认证，防伪造设备接入 |
| 入侵检测 | 网络流量监控，异常行为告警 |
| 防火墙 | 各网络边界部署工业防火墙 |
| 远程访问控制 | 远程访问需VPN/数字证书双因素认证 |

## 6.7 远程监控与操控

| 功能 | 说明 |
|---|---|
| 远程视频监控 | 实时查看4K视频流，多路同屏 |
| 远程操控 | 5G/光纤链路远程下发控制指令 |
| 远程配置 | 远程修改系统参数、切换车型模板 |
| 远程升级 | OTA远程推送系统升级包 |
| 远程故障诊断 | 远程查看日志、故障码、系统状态 |
