## 09 智能养护决策系统

### 一、智能养护决策逻辑

本系统以 AI 为驱动，基于大数据平台的分析结果，构建从检测到养护的完整闭环：

```
检测数据 ──→ 大数据分析 ──→ 养护优先级排序 ──→ 养护计划生成
                  │                                    │
                  ↓                                    ↓
            健康趋势预测           养护效果验证
                  │                                    │
                  ↓                                    ↓
           预测性维护建议         二次检测数据闭环
```

**决策链路说明：**

| 阶段 | 输入 | 处理 | 输出 |
|---|---|---|---|
| 检测采集 | 原始图像/点云/姿态 | AI 边缘推理 | 缺陷判定结果 |
| 大数据分析 | 历史+当前缺陷数据 | 健康评分/趋势分析 | 健康评估报告 |
| 优先级排序 | 健康评分+趋势+病害等级 | 多因子加权算法 | 优先级养护清单 |
| 计划生成 | 优先级清单+工时估算 | AI 决策模型 | 本周养护计划 |
| 效果验证 | 养护后二次检测数据 | 对比分析 | 效果评分 |

**AI决策的三大核心能力：**
- **数据驱动**：决策完全基于历史积累的检测数据，而非人工经验
- **量化优先**：每项建议均有量化依据（评分/趋势/工时），拒绝模糊表述
- **全链透明**：用户始终可知"为什么建议修这里"

### 二、养护优先级算法

系统采用多因子加权综合算法，对全线所有区段进行养护优先级排序：

**算法公式：**

$$
P = \alpha \times G + \beta \times T + \gamma \times L + \delta \times E
$$

其中：
- $P$：综合优先级得分（越高越优先处理）
- $G$：病害等级得分（当前病害的严重程度）
- $T$：劣化趋势得分（病害发展速度，越恶化越优先）
- $L$：线路重要度（重载/弯道/桥隧等优先）
- $E$：环境因子（气候/季节等因素对病害的影响）
- $\alpha, \beta, \gamma, \delta$：各因子权重，且 $\alpha+\beta+\gamma+\delta=1$

**各因子详细定义：**

**病害等级得分 $G$：**

| 病害等级 | $G$ 得分 |
|---|---|
| 三级缺陷 | 100 |
| 二级缺陷 | 60 |
| 一级缺陷 | 20 |

**劣化趋势得分 $T$（本次 vs 上次检测结果变化）：**

| 趋势变化 | $T$ 得分 |
|---|---|
| 缺陷数量大幅增加（>50%） | 100 |
| 缺陷数量增加（20%~50%） | 60 |
| 缺陷数量基本持平（±20%） | 30 |
| 缺陷数量减少 | 10 |
| 无历史数据（首次检测） | 40 |

**线路重要度 $L$：**

| 线路类型 | $L$ 得分 |
|---|---|
| 重载线路（轴重 > 25t） | 100 |
| 高速线路（设计时速 > 250km/h） | 90 |
| 桥隧连接段 | 80 |
| 道岔区域 | 70 |
| 普通线路 | 50 |

**权重配置（默认值）：**

| 因子 | 默认权重 | 说明 |
|---|---|---|
| $\alpha$（病害等级） | 0.40 | 病害严重程度是最重要因素 |
| $\beta$（劣化趋势） | 0.30 | 劣化速度次之，快速恶化需优先处理 |
| $\gamma$（线路重要度） | 0.20 | 重要线路优先保障 |
| $\delta$（环境因子） | 0.10 | 辅助调节 |

**优先级分档：**

| 优先级 | $P$ 得分 | 养护行动 |
|---|---|---|
| **P0 — 立即处理** | 80~100 | 24小时内处置，启动应急养护 |
| **P1 — 尽快计划** | 60~79 | 纳入本周计划，3~7天内处置 |
| **P2 — 重点关注** | 40~59 | 纳入本月计划，安排养护资源 |
| **P3 — 持续观察** | 20~39 | 缩短巡检周期，持续跟踪 |
| **P4 — 正常维护** | 0~19 | 纳入例行养护计划 |

**输出：** 自动生成"本周养护建议清单"，每条建议包含：里程区段 + 病害类型 + 等级 + 综合得分 + 建议措施 + 预估工时

### 三、AI 全说明决策支撑

本系统的每一项养护建议，均附带完整的决策依据，让用户真正理解"AI 为什么建议修这里"：

**决策依据包含以下四个维度：**

**1. 病害定位**
- 里程区段：KXXX+XXX ~ KXXX+XXX（精确到米）
- 病害位置：左轨/右轨 + 轨面具体位置描述
- 病害数量：该区段本次检测到的各类缺陷数量

**2. 判定依据**
- AI 识别依据：用于判定的图像区域截图 + AI 置信度
- 实测偏差数值：如磨耗量 X mm（标准值 Y mm，超限 Z mm）
- 历史对比趋势：该区段近 3 次检测的缺陷数量变化曲线
- 劣化速度：月均增加速率

**3. 养护建议**
- 标准依据：依据的养护规范（如《铁路线路维修规则》相关条款）
- 推荐措施：具体的养护工艺建议（如"打磨/更换/紧固"）
- 优先级说明：P几几分，为什么优先处理

**4. 预估工时**
- 基于病害类型和范围的工时估算
- 所需材料清单（如有）
- 影响行车的预计时间

**决策透明度示例：**

```
养护建议：K115+200 ~ K115+350 段钢轨裂纹

【病害定位】
里程：K115+200 ~ K115+350（共计 150m）
位置：左轨轨面内侧
本次检测：发现 3 处裂纹，最长 28mm，最深处 2.1mm

【判定依据】
AI 依据：图像分割区域置信度 96.3%，裂纹形态符合《铁路线路
         维修规则》中"局部碎裂"特征
偏差数值：实测深度 2.1mm（二级缺陷阈值 1.5mm），超限 0.6mm
历史对比：该区段上次检测（3个月前）无裂纹记录
劣化趋势：月均新增裂纹 1 处，劣化速度较快

【养护建议】
标准依据：《铁路线路维修规则》第 6.2.3 条
推荐措施：钢轨局部打磨或更换处理
优先级：P1（综合得分 72分）

【预估工时】
预计工时：4 小时（含天窗点）
影响范围：限速 45km/h 通过
```

> 用户可知晓"AI 为什么要建议现在修这里而不是别处"，决策过程全透明。每一项建议均可溯源、可解释、可审核。

### 四、养护效果量化验证

本系统建立了完整的养护效果量化验证机制，实现检测→养护→验证的闭环管理：

**闭环流程：**

```
养护执行 ──→ 二次检测 ──→ 数据对比 ──→ 效果量化评分
                                      │
                          效果达标 ←── └──→ 效果不达标→重新制定方案
```

**量化验证指标：**

| 指标 | 计算方式 | 达标阈值 |
|---|---|---|
| 缺陷消除率 | （养护前缺陷数 - 养护后缺陷数）/ 养护前缺陷数 × 100% | ≥ 90% |
| 缺陷复发率 | 养护后 3 个月内同位置再次出现同类型缺陷的比例 | ≤ 5% |
| 养护响应时间 | 养护完成到系统记录的时间间隔 | ≤ 24h |
| 评分提升值 | 养护后健康评分 - 养护前健康评分 | ≥ 15 分 |

**效果量化评分（0~100）：**

| 评分区间 | 效果等级 | 说明 |
|---|---|---|
| 90~100 | 优秀 | 缺陷完全消除或显著改善，养护措施精准有效 |
| 75~89 | 良好 | 缺陷大部分消除，偶有轻微复发 |
| 60~74 | 一般 | 缺陷有所减少但未达预期，需加强跟踪 |
| 40~59 | 较差 | 缺陷未明显减少或出现复发，需重新评估方案 |
| 0~39 | 失效 | 缺陷数量不降反升，养护方案需彻底重新制定 |

**闭环管理意义：**
- 打通了"检测→养护→验证"全链路，数据不断层
- 量化效果让养护质量可评价、可考核
- 复发数据反馈至 AI 模型，持续优化养护建议精度

### 五、养护计划自动生成与执行跟踪

**计划自动生成逻辑：**

| 输入 | 说明 |
|---|---|
| 优先级养护清单 | 系统按优先级排序的所有待养护区段 |
| 可用天窗点 | 线路天窗时间窗口（可接入工务系统数据） |
| 养护工时估算 | 每项建议的标准工时 |
| 资源约束 | 可用养护人员和设备数量 |

**计划生成算法：**
- 按优先级从高到低，优先将 P0/P1 排入最近可用天窗点
- 在天窗点时间内尽量排满，减少天窗浪费
- 冲突检测：同一区段时间窗重叠时，给出调整建议

**执行跟踪：**

| 状态 | 说明 |
|---|---|
| 待执行 | 已纳入计划，等待天窗点 |
| 执行中 | 当前正在养护作业 |
| 已完成 | 养护完成，数据已录入 |
| 验证中 | 等待下次检测验证效果 |
| 效果达标 | 二次检测确认效果达标 |
| 效果不达标 | 需重新制定养护方案 |

养护执行完成后，系统自动向相关人员推送完成通知，并将执行记录归档至历史数据库。
