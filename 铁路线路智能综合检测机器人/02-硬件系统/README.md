# 02-硬件系统

> **核心定位一句话：** 本硬件系统不是当前最先进传感器的简单堆叠，而是将工业面阵相机、3D线激光轮廓扫描、惯性测量单元（IMU）等多源感知技术进行有机融合，组成一台精密的综合检测设备，代表当下钢轨线路检测领域硬件系统的最高集成水平。
>
> **三层精密融合：**
> - **传感器层融合**：不同类型传感器数据时间戳对齐、坐标系统一，多视角互相校验
> - **通信层融合**： EtherCAT实时控制总线 + 千兆工业以太网数据传输 + 4G/5G无线上传，三网协同、互不干扰
> - **数据层融合**：2D图像 + 3D点云 + 姿态参数在统一框架下联合分析，输出综合性检测结论
>
> 本目录所有硬件选型、设计文档、电气电路均围绕这一核心目标——以最先进的传感器组合，实现综合检测能力质的飞跃。

## 二、运动平台

### 2.1 机械结构

- **行驶方式**：4个轮子直接骑跨在钢轨上，跟随牵引系统移动
- **轮组布局**：前后各2个轮对，共4个主动轮
- **驱动方式**：4个伺服电机独立驱动，精确控制速度与位置
- **运行速度**：1 m/s（1000 mm/s）稳定匀速
- **运动控制协议**：EtherCAT总线，实时性强

### 2.2 运动控制系统

| 项目 | 参数/规格 |
|-----|---------|
| 控制协议 | EtherCAT |
| 伺服电机数量 | 4个（每个轮组独立驱动） |
| 电机类型 | 伺服电机（高动态响应） |
| 控制功能 | 启停控制 / 速度调节 / 姿态保持 / 运动协调 |
| 安全机制 | 失速保护 / 过载保护 / 急停联锁 |

### 2.3 行驶机构设计原则

- 车轮与钢轨的接触几何经过设计，确保在运行过程中不脱轨
- 传感器安装基座与车轮基座刚性连接，保证传感器与轨道几何关系的稳定性
- 小车整体重心低，行驶过程中姿态稳定，不影响传感器采集质量

## 三、感知系统

感知系统是综合检测的核心输入端，由三类传感器组成，共同构成多源感知融合的硬件基础。

### 3.1 2D工业相机（6个）

所有相机统一采用**垂直朝下**安装方式，覆盖轨面和螺栓区域。

| 相机编号 | 覆盖区域 | 主要检测对象 | 输出 |
|---------|---------|------------|------|
| CAM1 | 左轨轨面 | 左轨轨面磨损、鱼鳞纹、脱落、表面异常 | 原始图像 + 异常图像 + 检测结果 |
| CAM2 | 右轨轨面 | 右轨轨面磨损、鱼鳞纹、脱落、表面异常 | 原始图像 + 异常图像 + 检测结果 |
| CAM3 | 左轨螺栓区域 | 左轨螺栓完好状态、缺失、松动、异常 | 原始图像 + 螺栓状态结果 |
| CAM4 | 右轨螺栓区域 | 右轨螺栓完好状态、缺失、松动、异常 | 原始图像 + 螺栓状态结果 |
| CAM5 | 左轨补充区域 | 左轨轨面及螺栓补充复核采集 | 补充图像 + 复核图像 |
| CAM6 | 右轨补充区域 | 右轨轨面及螺栓补充复核采集 | 补充图像 + 复核图像 |

**相机关键参数：**

| 项目 | 参数 |
|-----|------|
| 分辨率 | 2448×2048（500万像素级面阵） |
| 镜头焦距 | 8mm / 12mm（根据覆盖区域选型） |
| 安装高度 | 轨面检测相机距轨面约 180mm；螺栓检测相机距轨面约 270mm |
| 安装方式 | 垂直朝下，不采用侧向视角 |
| 供电方式 | PoE（以太网供电） |
| 通信接口 | 千兆工业以太网（GigE Vision协议） |

**相机帧率与触发参数（定稿）：**

| 相机 | 触发模式 | 目标帧率 | 曝光上限 | ROI策略 |
|-----|---------|---------|---------|---------|
| CAM1（左轨面） | 连续采集 | 20 fps | ≤800µs | 轨面主ROI |
| CAM2（右轨面） | 连续采集 | 20 fps | ≤800µs | 轨面主ROI |
| CAM3（左螺栓区） | 里程触发为主 + 连续兜底 | 25 fps | ≤700µs | 螺栓双侧ROI |
| CAM4（右螺栓区） | 里程触发为主 + 连续兜底 | 25 fps | ≤700µs | 螺栓双侧ROI |
| CAM5（左补充复核） | 连续采集（复核优先） | 20 fps | ≤800µs | 轨面+螺栓补充ROI |
| CAM6（右补充复核） | 连续采集（复核优先） | 20 fps | ≤800µs | 轨面+螺栓补充ROI |

> **同步策略：** 以 Frame ID + 里程脉冲为同步主键，所有相机数据按 Frame ID 对齐入库，保证多相机数据在同里程位置严格同步，不错位、不乱序。
> **验收标准：** 全程帧率不低于设定值，连续运行丢帧率接近0，螺栓边缘无明显拖影，轨面纹理可判读。

> **综合检测视角：** 6个相机不是各自为战，而是统一规划覆盖区域——左右轨面各有专属相机，螺栓区域各有专属相机，补充区域用于重点复核。数据同源、同一时刻采集，为后续多维数据融合分析提供硬件基础。

### 3.2 3D线激光（2个）

**统一采用垂直朝下安装**，分别检测左右两个钢轨的轨头轮廓。

| 激光编号 | 检测对象 | 功能定位 |
|---------|---------|---------|
| 激光1 | 左钢轨轨头轮廓 | 左轨三维廓形检测 |
| 激光2 | 右钢轨轨头轮廓 | 右轨三维廓形检测 |

**工作原理：**
- 激光器主动向钢轨表面投射线性光条
- 相机从特定角度拍摄光条形变
- 三角测量法还原轨头三维截面形状
- 与标准廓形模板比对，计算磨耗量、横移量

**关键特性：**

| 项目 | 说明 |
|-----|------|
| 主动测量 | 不依赖外部光源，消除环境光干扰 |
| 垂直朝下 | 简化安装几何，保证测量精度一致性 |
| 实时轮廓输出 | 每帧输出轨头横截面点云数据 |

> **综合检测视角：** 3D线激光是综合检测中"三维感知"能力的核心担当。与2D相机数据时间戳对齐后，可以将轨面的二维图像缺陷与三维廓形变化联合分析——例如，同一处既有表面裂纹（2D图像检出）又有局部变形（3D轮廓检出），两种数据互相印证，判级更准确。

### 3.3 姿态参数传感器（2个陀螺仪）

**2个姿态参数传感器，专门用于检测钢轨线路的几何姿态参数。**

| 传感器 | 检测参数 | 说明 |
|-------|---------|------|
| 陀螺仪1 | 水平（方向）、高低、轨向 | 姿态角实时解算 |
| 陀螺仪2 | 姿态冗余备份 | 双传感器备份，提高可靠性 |

**检测内容：**

| 参数 | 说明 | 对应检测项目 |
|-----|------|------------|
| 水平 | 左右钢轨高差 | 水平度检测 |
| 高低 | 沿线路方向的垂直不平顺 | 高低不平顺检测 |
| 轨向 | 沿线路方向的水平偏差 | 轨向检测 |

> **综合检测视角：** 陀螺仪数据与图像/激光数据时间戳同步后，可以建立"缺陷位置 ↔ 姿态几何参数 ↔ 图像缺陷"的三维关联分析。例如，某处轨向偏差大，同时轨面裂纹集中出现在同一位——这种关联分析是单一功能检测设备做不到的。

### 3.4 传感器数据同步机制

| 项目 | 规格 |
|-----|------|
| 同步精度 | 毫秒级时间戳对齐 |
| 同步方式 | 统一触发信号，所有传感器同步采集 |
| 数据格式 | 统一坐标系统，里程+时间戳双重标记 |

| 同时处理路数 | 6相机 + 2激光 + 2陀螺仪全通量实时处理 |

### 4.2 数据总线与带宽估算

综合检测的多传感器同时工作，数据通量较大，系统设计需考虑总带宽：

| 数据源 | 数据规格 | 估算带宽 |
|-------|---------|---------|
| 6×2D工业相机 | 2448×2048 @ 20~25fps，GigE RAW | 约 1.5~2 Gbps（6路并发） |
| 2×3D线激光 | 帧率~XX fps，点云数据 | [待确认] |
| 2×陀螺仪 | 姿态角输出，数据量小 | < 10 Mbps |
| **合计** | | **约 2~3 Gbps 峰值** |

> **说明：** 实际带宽取决于帧率配置、ROI裁切比例、压缩策略（是否在相机端压缩）。工控机网卡和交换机需按峰值带宽的1.5~2倍冗余选型。

### 4.3 急停硬件保护回路

急停是物理世界的最后一道安全防线，必须完全独立于软件系统，硬件故障时也能可靠动作。

**急停回路设计原则：**

| 设计要点 | 说明 |
|---------|------|
| 硬线独立 | 急停信号走独立硬线回路，不经工控机、不经任何软件判断 |
| 断电即停 | 急停触发 → 继电器切断伺服驱动器使能 → 车轮立即抱死 |
| 物理按钮 | 金属实体急停按钮，分布在小车前后侧，现场人员可及 |
| 失效导向安全 | 继电器失电时默认停车（fail-safe），而非继续运行 |
| 报警联动 | 急停触发时，工控机同步收到硬线中断信号，记录急停事件日志 |

**急停回路拓扑：**

```
急停按钮（物理硬线） ──▶  安全继电器 ──▶  伺服驱动器使能信号
                                  │
                                  ▼
                          工控机中断信号（记录日志）
```

### 4.5 通信与数据传输

| 项目 | 说明 |
|-----|------|
| 有线通信 | 千兆工业以太网（相机/激光/工控机之间） |
| 无线通信 | 4G/5G工业路由器（工控机→云端） |
| 上传策略 | 分级上传（P0急停告警→P1核心结果→P2关键图像→P3原始数据批量） |
| 本地存储 | SSD，本地缓存，网络中断时断点续传 |

> **综合检测视角：** 综合检测产生的数据量远大于单一功能检测，分级上传策略确保最重要的数据优先到达云端，一般数据本地缓存后批量上传，不丢数据也不堵塞链路。

## 六、人机交互系统

人机交互系统提供完整的操作控制能力，详见项目总体介绍文档。

### 三种控制方式

| 控制方式 | 实现 | 适用场景 |
|---------|------|---------|
| 金属物理按钮 | 实体按键，盲操作可靠 | 现场紧急操作/屏幕故障兜底 |
| 触摸屏控制 | 参数设置+状态监控+数据预览 | 出发前设置/作业中监控 |
| 遥控控制 | 远距离控制启停、速度、返程 | 复杂现场/远距离操作 |

### 智能驾驶辅助

- **定速巡航**：设定速度后自动保持匀速，采集速度一致，数据质量稳定
- **定点返程**：一键自动返航至起点，防止复杂现场无法回收

### 控制通信可靠性

- 心跳信号 / 指令确认（Ack）/ 超时保护
- 失效安全（Fail-Safe）/ 双通道备份 / 看门狗（Watchdog）
- 详细机制见项目总体介绍文档

## 七、硬件系统与综合检测的对应关系

| 综合检测功能 | 硬件支撑 |
|------------|---------|
| 轨面缺陷检测 | CAM1~CAM6（轨面覆盖） |
| 螺栓/道钉检测 | CAM3~CAM4（螺栓区域） |
| 3D轨头轮廓检测 | 2个3D线激光 |
| 轨距检测 | 传感器融合计算 |
| 水平/高低/轨向 | 2个陀螺仪 |

| 实时数据上传 | 工控机+4G/5G工业路由器 |
| 用电安全监控 | 48V智能供电系统+电池BMS |

> **结论：** 硬件系统不是各子模块的简单拼装，而是围绕综合检测这一核心目标进行的系统化设计。每一种传感器、每一级供电、每一路通信，都服务于"同时采集多种参数→统一分析→综合输出"这一完整检测链条。

## 附：硬件系统的三层精密融合

本节将综合检测背后的硬件融合逻辑集中阐述，作为全系统的核心技术注脚。

### 附1. 传感器层有机融合（不是加法，是化学反应）

| 融合维度 | 融合方式 | 融合效果 |
|---------|---------|---------|
| 2D + 3D | 同一位置同时有相机图像和激光轮廓，互为印证 | 表面裂纹 + 形变同时检出，误报率降低 |
| 图像 + 姿态 | 缺陷图像绑定对应位置的陀螺仪姿态角 | 知道"这儿有裂纹"的同时知道"这儿的轨向也在偏" |
| 左右轨对称 | 左/右轨各有独立相机+激光，对称布局 | 左右轨缺陷可对比分析，发现不对称劣化趋势 |
| 主检 + 复核 | CAM1~CAM4主检 + CAM5~CAM6复核 | 重点区域双确认，重要缺陷不漏报 |

**不是"每种传感器各采各的"，而是"所有传感器在同一时空框架下协同工作，输出统一结论"。**

### 附2. 通信层精密协同（三网融合、互不干扰）

```
┌──────────────────────────────────────────────────────────────┐
│                    三网物理隔离通信架构                        │
│                                                              │
│  实时控制网（EtherCAT）  ── 毫秒级硬实时 ── 伺服启停/急停    │
│       │                                                       │
│  数据采集网（千兆以太网）── 软实时       ── 相机/激光数据回传  │
│       │                                                       │
│  外部传输网（4G/5G）    ── 非实时       ── 云端上传/远程监控  │
│                                                              │
│  三张网物理隔离，带宽独立，确保控制指令永不被数据流量堵塞       │
└──────────────────────────────────────────────────────────────┘
```

### 附3. 数据层统一分析框架

```
统一时空框架（Frame ID + 里程 + 时间戳）
       │
       ├── 2D图像流 ──▶ AI缺陷识别 ──┐
       ├── 3D点云流 ──▶ 廓形比对   ──┼──▶ 融合分析 → 综合判级
       └── 姿态参数流 ──▶ 几何计算 ──┘
```

> **核心价值：** 传感器是当代最先进的，数据融合不是简单叠加，是精密时空对齐后的有机配合——这才是综合检测硬件系统真正的技术门槛所在。
