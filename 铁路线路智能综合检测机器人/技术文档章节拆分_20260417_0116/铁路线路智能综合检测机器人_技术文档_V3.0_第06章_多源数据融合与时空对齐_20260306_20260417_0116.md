## 06 多源数据融合与时空对齐

### 一、三流数据融合架构

本系统同时采集三路数据流，在统一时空框架下融合分析：

```
┌─────────────────────────────────────────────────────────────────┐
│                     边缘计算层工控机（融合节点）                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐       │
│  │ 2D工业相机×6 │    │3D线激光×2   │    │单点测距×2   │       │
│  │  图像流      │    │  廓形流      │    │  高程流      │       │
│  └──────┬───────┘    └──────┬───────┘    └──────┬───────┘       │
│         │                   │                   │                 │
│         ▼                   ▼                   ▼                 │
│  AI 缺陷识别          廓形拟合/波磨分析     EKF 姿态融合          │
│  （轨面/道钉/焊缝）    （独立链路）         + HWT905 IMU          │
│         │                                        │                 │
│         └────────────────┬──────────────────────┘                 │
│                          ▼                                        │
│                   AI 综合判级                                      │
│                          │                                        │
│                          ▼                                        │
│                   输出结论（里程/缺陷类型/等级）                       │
└─────────────────────────────────────────────────────────────────┘
```

### 二、时钟同步机制

| 技术方案 | 说明 |
|---|---|
| 统一时间戳 | 所有传感器共享同一时钟源，毫秒级同步 |
| 帧编号对齐 | 每帧图像/点云/姿态数据对应唯一编号 |
| 里程坐标对齐 | 所有数据绑定同一里程坐标，位置唯一 |
| 触发同步 | 相机/单点测距传感器/HWT905同步触发采集，数据同源 |

### 三、传感器层有机融合

- **2D 图像 + 3D 廓形**：同位置同步采集，缺陷与形变互相印证，降低误报率
- **单点测距 + HWT905 IMU**：EKF融合，姿态角实时校正测距值，输出轨道高程（见§4.4.1）
- **图像数据 + 姿态参数**：缺陷位置绑定姿态角，定位更精准
- **左右钢轨双侧独立检测**：对称采集、独立判定、结果互补
- **主检 + 复核机制**：重点区段双重采集、双重确认，降低漏检

### 四、数据层统一分析

- **三码对齐**：帧编号 + 里程 + 时间戳，数据同源可追溯
- **图像流处理**：2D 图像流 → AI 缺陷智能识别
- **廓形流处理**：3D 点云流 → 廓形拟合 → 磨耗/波磨分析
- **融合判级**：三流数据汇入融合节点，综合判级，输出最终结论

### 五、多链数据融合详细算法实现

本系统有三条独立的数据处理链路，各链路传感器配置、采样率、融合算法均不同，最终在AI综合判级层汇流：

| 链路 | 主传感器 | 辅传感器 | 融合算法 | 输出 |
|---|---|---|---|---|
| **链路A：AI视觉缺陷检测** | 2D工业相机×6 | HWT905（定位校正）| AI图像分类 | 缺陷类型+等级+里程 |
| **链路B：廓形/波磨分析** | 3D线激光×2 | — | 廓形拟合+FFT频谱 | 磨耗量+波深+波长 |
| **链路C：高程融合（EKF）** | 单点测距传感器×2 | HWT905 | EKF最优估计 | 轨道高低不平顺值 |

> **链路C即§4.4.1所述EKF融合算法，在本章中聚焦三条链路的并行处理与最终汇流逻辑。**

**第一级：时间对齐（Temporal Alignment）**

- 统一时钟源：GPS 秒脉冲（PPS）或 IEEE 1588 PTP 精确时间协议，精度 ±0.1ms
- HWT905 IMU数据：200Hz 采样，内插至与相机帧同步（25fps）
- 里程编码器：每 1mm 产生一个里程脉冲，绑定到最近帧
- 融合窗口：50ms 滑动窗口，窗口内所有传感器数据标记为同一时刻

**第二级：空间关联（Spatial Association）**

- 左右轨独立建模：左右轨各有独立的图像流 + 点云流 + 姿态流
- 位置坐标 = 里程坐标（X轴） + 轨距偏移（Y轴） + 高度偏移（Z轴）
- 缺陷位置 = 里程（X）+ 左右轨标识 + 轨内偏移
- 姿态角修正：缺陷位置根据俯仰角/侧倾角实时修正，消除颠簸导致的定位误差

**第三级：判级融合（Decision Fusion）**

采用加权投票机制，根据不同缺陷类型分配不同权重：

| 缺陷类型 | 图像权重 | 点云权重 | 姿态权重 | 融合逻辑 |
|---|---|---|---|---|
| 轨面裂纹 | 70% | 20% | 10% | 图像为主，轮廓印证 |
| 轨面掉块 | 60% | 30% | 10% | 双维度同时超限才判定 |
| 螺栓松动 | 65% | 10% | 25% | 图像为主，振动辅助 |
| 轮廓磨耗 | 10% | 90% | 0% | 点云独立判定 |
| 波磨 | 20% | 80% | 0% | 点云频谱分析为主 |
| 焊缝缺陷 | 80% | 15% | 5% | 图像为主 |

### 六、时钟同步技术细节

本系统采用双时钟源冗余同步机制，确保全系统时间一致性：

**主时钟源：GPS/北斗 PPS（秒脉冲）**

| 项目 | 指标 |
|---|---|
| 同步精度 | ±0.1ms（相对于UTC） |
| 有效条件 | 户外或天空视野良好 |
| 备份机制 | GNSS 信号丢失时自动切换至 PTP |

**备份时钟源：IEEE 1588 PTP（精确时间协议）**

| 项目 | 指标 |
|---|---|
| 同步精度 | ±0.5ms（局域网内） |
| 适用场景 | GNSS 信号不可用（隧道/室内/遮蔽） |
| 交换机构成 | 支持 PTP 的工业以太网交换机 |

**各传感器时钟同步方式：**

| 传感器 | 同步方式 | 同步精度 |
|---|---|---|
| 工业相机（6路） | GigE Vision 协议内嵌时间戳 + 外部触发信号 | ±0.5ms |
| 3D 线激光（2路） | TCP/IP 时间同步 + 外部触发信号 | ±1ms |
| HWT905 IMU | UART 直连，200Hz 采样内插 | ±1ms |
| 里程编码器 | 计数器直连，1mm/脉冲 | 物理精度 ±1mm |
| 工控机 | NTP 同步至主时钟源 | ±5ms |

### 七、传感器数据对齐精度

综合各传感器融合后的最终数据精度：

| 数据类型 | 里程精度 | 横向精度 | 垂向精度 | 时间精度 |
|---|---|---|---|---|
| 轨面缺陷位置 | ±5mm | ±2mm | ±1mm | ±1ms |
| 道钉/螺栓位置 | ±5mm | ±2mm | ±2mm | ±1ms |
| 3D 轮廓数据 | ±5mm | ±0.5mm | ±0.1mm | ±2ms |
| 波磨数据 | ±5mm | ±1mm | ±0.1mm | ±2ms |
| 轨距值 | ±2mm | — | — | ±1ms |
| 高低不平顺（EKF输出）| ±5mm | — | ±0.8mm | ±2ms |
| 水平/高低角（HWT905）| — | — | ±0.01° | ±2ms |

**精度保障措施：**

- 每帧数据带有质量标志（Good/Bad/Uncertain），坏数据不参与融合
- 丢帧自动补齐：前帧数据时间戳延长，避免融合流水线断裂
- 异常值剔除：3σ 原则剔除突变数据点

### 八、实时融合性能指标

| 指标 | 数值 | 说明 |
|---|---|---|
| 融合处理延迟 | ≤ 10ms | 从接收到输出融合结果 |
| 最大并发处理帧数 | 8 帧 | 8 路传感器并行融合 |
| 融合吞吐量 | 400 帧/秒 | 满足 6相机+2激光+2单点测距 满载 |
| 系统可用性 | ≥ 99.9% | 融合节点故障不影响本地独立判定 |
| 内存占用 | ≤ 8GB | 50ms 滑动窗口数据缓存 |

### 九、数据质量保障机制

| 措施 | 说明 |
|---|---|
| 帧质量标志 | 每帧图像附带曝光/增益/帧率质量评分，低质量帧标记不参与融合 |
| 丢帧检测 | 连续 3 帧丢失触发告警，启动丢帧补偿（时间戳延长） |
| 异常值剔除 | 每个数据流独立做 3σ 统计，突变点剔除并标记 |
| 数据校验和 | 帧数据 CRC32 校验，损坏数据不进入融合流水线 |
| 传感器健康监测 | 传感器帧率/温度/电压实时监测，异常传感器数据降权或屏蔽 |

### 十、左右轨独立融合判定流程

左右轨数据完全独立处理，互不干扰，最终结果统一输出：

```
左轨数据流                                                          右轨数据流
   │                                                                    │
   ▼                                                                    ▼
左轨图像帧 ──┐                                                     右轨图像帧 ──┐
左轨廓形帧 ──┼──→ 左轨融合节点 ──→ 左轨判定结果                 右轨廓形帧 ──┼──→ 右轨融合节点 ──→ 右轨判定结果
左轨高程帧 ──┘                                                     右轨高程帧 ──┘
   │                                                                    │
   └────────────────────────────┬───────────────────────────────────────┘
                                ▼
                         双轨结果汇总
                                │
              ┌─────────────────┴──────────────────┐
              ▼                                   ▼
    AI综合判级（缺陷类）              EKF输出（高低不平顺TQI）
              │                                   │
              └─────────────────┬─────────────────┘
                                ▼
                         最终输出（含里程/左右轨/缺陷等级/轨道质量指数）
```

**双轨独立判定的好处：**

- 左右轨缺陷互不影响，避免误判传播
- 单轨传感器故障时，另一轨可独立继续工作
- 方便左右轨缺陷分开统计和养护
