## 08 线路健康检测平台

### 一、大数据平台定位

本系统的线路健康检测平台是整个检测体系的数据中枢和价值出口：

> **检测数据 → 实时汇聚 → 积累沉淀 → 挖掘分析 → 决策支撑**

**平台核心定位：**
- **数据中枢**：汇集每一次检测任务产生的全量数据，实现数据的统一存储、统一管理、统一服务
- **分析引擎**：基于历史积累的海量数据，提供趋势分析、劣化预测、健康评估等高级分析能力
- **决策支撑**：为养护决策提供数据依据，让每一项养护建议都有数据支撑
- **可视化窗口**：将复杂的检测数据和健康状态以直观的方式呈现给不同用户

**平台架构设计：**

```
检测机器人（边缘） ──实时数据回传──→ 数据接入层 ──→ 数据存储层
                                                    │
                     云端大数据平台 ←───────────────┘
                          │
          ┌──────────────┼──────────────┐
          ↓              ↓              ↓
      数据服务        分析服务       展示服务
   （检索/导出）   （评分/预测）   （热力图/报表）
```

**平台价值总结：**
没有大数据，就没有线路健康分析、没有趋势预测、没有科学的养护决策。

### 二、平台数据来源

平台数据来源于本系统三大核心数据流，覆盖检测全流程：

| 数据来源 | 数据内容 | 数据量 | 更新频率 | 用途 |
|---|---|---|---|---|
| 每次检测任务 | 原始图像流 + 点云流 + 姿态流 + AI判定结果 | ~100GB/公里 | 每任务结束后批量上传 | 核心分析数据 |
| 实时缺陷数据 | 缺陷类型 + 里程位置 + 损伤等级 + 置信度 | MB级/公里 | 实时推送 | 实时告警和展示 |
| 设备运行日志 | 相机帧率 / 激光温度 / 电池电量 / 网络状态 | GB级/月 | 秒级 | 设备健康监控 |
| 养护维修记录 | 养护时间 / 区段 / 内容 / 执行方 / 验证结果 | MB级/次 | 人工录入 | 养护效果闭环 |
| 历史累积数据 | 多轮次同一区段的检测档案 | 持续增长 | 累积 | 趋势分析和劣化预测 |
| 环境关联数据 | 温度 / 湿度 / 气候条件（可选接入） | 关联分析用 | 实时/历史 | 与缺陷关联分析 |

**数据安全与权限：**
- 检测任务数据归属客户方，平台方仅提供技术服务
- 数据传输全程 TLS 1.3 加密
- 云端存储多副本容灾，持久性 99.999999999%
- 不同角色（操作员/管理员/决策者）数据可见范围可配置

### 三、线路健康检测平台

#### 3.1 数据汇聚与存储管理

**多任务数据自动汇聚：**
- 每次检测任务结束后，数据自动从边缘设备上传至云端大数据平台
- 多任务数据按"线路 → 区段 → 日期"三级目录组织
- 同一区段多轮次检测数据自动归档至同一目录下，便于横向对比

**时空索引与多维检索：**
- 所有数据建立时空索引，支持以下检索维度：
  - 按里程范围（公里标）：精确到米
  - 按时间区间：任意时间段
  - 按病害类型：裂纹/掉块/松动/磨耗等
  - 按健康评分：0~100分任意区间
  - 组合检索：里程+时间+病害类型同时筛选

**历史数据回放：**
- 任意历史任务的原始图像、点云、判定结果可随时调取
- 支持指定里程点的多轮次数据横向对比（同一位置不同时间）

#### 3.2 GIS线路健康可视化

平台以电子地图为底座，叠加检测数据，实现全线健康状况一张图：

**地图底图：**
- 接入铁路线路电子地图（工务GIS系统或第三方地图）
- 支持矢量地图和卫星影像两种底图模式

**数据叠加层：**

| 叠加层 | 内容 | 显示方式 |
|---|---|---|
| 轨迹层 | 检测机器人行驶轨迹 | 蓝色线条 |
| 缺陷层 | 各类型缺陷位置标注 | 按等级颜色标注（1级绿/2级黄/3级红） |
| 热力层 | 线路健康综合评分 | 绿~黄~红渐变色带 |
| 养护层 | 历史养护区段标记 | 紫色标记 |

**操作交互：**
- 点击任意里程点 → 弹出该点详细信息（缺陷类型/等级/历史记录/养护状态）
- 拖动时间轴 → 回放任意时间段内的检测数据变化
- 框选区段 → 批量导出选中区域的检测报告

#### 3.3 健康综合评分算法

平台对全线各区段给出 0~100 的综合健康评分，评分依据各病害的加权计算：

**评分公式：**

$$
S = 100 - \sum_{i=1}^{n} (W_i \times D_i \times L_i)
$$

其中：
- $S$：区段健康评分（0~100）
- $W_i$：第 $i$ 种病害类型的权重系数
- $D_i$：第 $i$ 种病害的严重程度系数（1~3，对应一/二/三级缺陷）
- $L_i$：该病害在区段内的分布密度（缺陷数量/公里）

**病害权重系数（示例）：**

| 病害类型 | 权重 $W_i$ | 说明 |
|---|---|---|
| 钢轨裂纹 | 15 | 高速安全风险高 |
| 钢轨掉块 | 12 | 影响列车平稳性 |
| 道钉缺失 | 5 | 影响轨道稳定 |
| 道钉松动 | 3 | 隐患逐步扩大 |
| 轮廓磨耗超限 | 8 | 影响轮轨关系 |
| 波磨超限 | 6 | 影响运行平稳性 |
| 焊缝缺陷 | 10 | 重点薄弱环节 |

**区段划分原则：**
- 按固定长度区段评分（如每100米一个评分单元）
- 可按实际里程标记（如咽喉区、桥头、隧道口等）自定义区段

**评分展示：**
- GIS地图上以颜色渐变（绿→黄→红）直观显示各区段健康状态
- 低于60分的区段自动列入养护建议候选

#### 3.4 趋势分析与劣化预测

**历史对比分析：**
- 同一区段多轮次检测数据自动对齐，生成缺陷数量变化曲线
- 重点关注：同一位置缺陷从无到有、从轻到重的劣化过程
- 自动标记劣化加速区段（本次缺陷数量环比显著增加）

**AI劣化预测模型：**
- 基于历史数据训练时序预测模型（ARIMA或LSTM）
- 输出预测结果：

| 预测周期 | 内容 |
|---|---|
| 1个月预测 | 病害数量和分布范围预测 |
| 3个月预测 | 重点劣化区段预警 |
| 6个月预测 | 养护优先级排序参考 |

**预测价值：**
将养护从"出了问题再修"转变为"看到趋势提前修"，实现从响应式维护到预防性维护的升级。

#### 3.5 多任务管理与数据检索

**任务管理列表：**

| 功能 | 说明 |
|---|---|
| 任务列表 | 显示所有历史任务，含起止里程/日期/完成状态 |
| 任务对比 | 选中2个以上任务横向对比健康评分和缺陷数量变化 |
| 任务归档 | 已完成任务数据永久保存，支持随时调取 |

**数据检索示例：**
- "K100+000 ~ K105+000 范围内，近6个月内二级以上缺陷有哪些？"
- "全线路当前评分低于70分的区段有哪些，按评分升序排列"
- "某个道岔区段，历史上共进行过几次养护，最近一次是什么时候"

#### 3.6 报表自动生成

平台支持一键生成标准检测报告，报告内容覆盖完整：

**报告内容：**

| 报告章节 | 内容 |
|---|---|
| 任务概况 | 检测里程/日期/设备状态/天气条件 |
| 缺陷总览 | 各类型缺陷数量统计/分布图 |
| 重点缺陷 | 二级及以上缺陷明细（含里程/照片/等级） |
| 健康评分 | 各区段评分热力图/评分表 |
| 趋势分析 | 重点区段劣化趋势（与历史对比） |
| 养护建议 | 基于评分和趋势的优先级养护清单 |

**报告格式与导出：**
- 支持导出 Word（.docx）和 PDF 两种格式
- 报告自动编号，可按日期/线路/区段归档
- 养护建议章节可直接提交至工务养护管理系统

**报表自动推送：**
- 任务结束后自动生成报告，推送至指定人员邮箱
- 支持设置周期性汇总报告（如每月/每季度自动汇总）
