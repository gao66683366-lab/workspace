## 晚间补充 (21:15)

### Word排版核心发现

**OOXML格式修改导致文字丢失的bug（之前一直没发现）**：
- 当用 `etree.remove(tcPr)` + `tc.insert(0, tcPr_new)` 替换单元格属性后，再通过 python-docx 的 `cell.paragraphs[0].text` 读取文字会返回空字符串
- **原因**：`insert()` 把新 tcPr 插到 tc 第一个子元素位置，但原 `<w:p>` 段落元素是第一个child，insert 后段落元素的 position 被打乱，python-docx 的 paragraph 代理对象失效
- **正确做法**：从 XML 直接解析文字（遍历 `tc.findall('w:p')` → `r.find('w:t')`），而不是通过 python-docx paragraph API 中转
- **或者**：先保存文字，再做 XML 操作，最后用 `etree.SubElement(p_el, 'w:r')` 直接写入

**表1列宽的经验值**：
- 表1（4列传感器参数）：总宽14cm=7938twips，`[3119, 1418, 1418, 1984]`
- 均分宽度会导致文字多的列被压缩，内容挤作一团
- 应按内容比例分配：传感器名最长→5.5cm，数值列→2.5cm，功能→3.5cm

**python-docx表格格式修改流程**（正确顺序）：
1. 从XML直接提取文字 `cell_text = ''.join(t.text or '' for r in p_el.findall('w:r') for t = r.find('w:t') if t is not None)`
2. 重建 tcPr（remove旧→insert新）
3. 重建段落（清空→centered pPr→单个带文字的run）
4. 设置 tblW 和 tblBorders

## 排版研究结论（重要）

### 原始文档问题诊断
- 原始文件无样式、无边框、无显式列宽 → Word/WPS用默认方式渲染，显示混乱
- 根本原因：document.xml里表格缺少完整的格式化属性

### Word表格格式三要素（必须同时满足）
1. **tblBorders** - 表格级别边框（Word/WPS优先读取）
2. **tcBorders** - 单元格级别边框（双重保险）
3. **tblGrid/gridCol** - 列宽定义（Word优先读取，决定列宽）
4. **tcPr/tcW** - 单元格宽度（python-docx API只改这个）

### 列宽正确设置方法
- `tblGrid/gridCol/w:w` + `tcPr/tcW/w:w` 必须同时设置且值相同
- Word 优先读 `tblGrid`，python-docx的`cell.width`只改`tcW`
- 正确做法：直接操作XML，同时写两个地方

### Word计数 vs WPS计数
- WPS计数 = 中文字符 + 英文字母 + 阿拉伯数字（不含空格）
- 用户要求：以后论文字数以WPS格式计数为准
- 本次精炼版：WPS显示7052，python-docx原始10251（差异来自空格和标点计数方式）
