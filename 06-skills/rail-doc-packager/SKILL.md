# Rail Doc Packager

## 输出目标
将零散技术结论整理为可直接落地的文档结构，默认输出为 Word 文档内容骨架（章节+表格字段）。

## 固定章节模板
1. 编制说明
2. 输入边界条件
3. 计算方法与公式
4. 参数取值与工程假设
5. CAM1~CAM6参数表
6. 触发与同步策略
7. 验收标准与测试记录
8. 结论与后续优化项

## 参数表最低字段
- 相机编号（CAM1~CAM6）
- 检测对象（轨面/螺栓/补充）
- 触发模式（连续/里程触发/混合）
- 目标帧率（fps）
- 曝光上限（us）
- 目标重叠率
- ROI配置
- ResultingFrameRateAbs验收下限
- 丢帧率阈值
- 拖影/清晰度判据

## 文档口径（当前项目）
- 轨头轨面宽度：70.8 mm（工程计算可取72 mm）
- 最大运行速度：3.6 km/h（1.0 m/s）
- 所有定稿文档需带日期后缀（如 `_2026-03-08`）

## 使用规则
- 先给出“参数草案表”，再给“定稿版表格”。
- 对每个关键参数标注来源：实测 / 假设 / 经验。
- 如参数冲突，优先保留“系统强约束”和用户最新口径。
