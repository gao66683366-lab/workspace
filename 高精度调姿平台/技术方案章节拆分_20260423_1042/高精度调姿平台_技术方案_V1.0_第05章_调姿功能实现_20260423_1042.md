# 5. 调姿功能实现

## 5.1 功能概述

基于单点测距传感器的轨道高程实时检测，结合Stewart平台的快速响应能力，实现行进过程中平台姿态的全自动、高精度、实时补偿。

## 5.2 功能原理

### 5.2.1 核心工作流程

| 步骤 | 内容 | 耗时 |
|---|---|---|
| 1. 高程测量 | 单点测距传感器以2000Hz采样轨道表面 | 0.5 ms |
| 2. 姿态解算 | 双传感器差值计算轨道横滚角、俯仰角 | 0.5 ms |
| 3. EKF融合 | 融合测距数据与HWT905姿态数据，输出真值 | 1.0 ms |
| 4. 逆运动学 | 目标姿态（水平）反解6缸目标长度 | 0.5 ms |
| 5. 指令下发 | EtherCAT总线发送至6轴伺服驱动器 | 0.5 ms |
| 6. 伺服响应 | 伺服驱动器接收并执行指令 | 1.0 ms |
| 7. 机械响应 | 电动缸伸缩，平台补偿至水平 | 1.0 ms |
| **全链路总计** | | **≤ 5 ms** |

### 5.2.2 姿态保持判定

平台"永远保持水平"的判定条件：

```
|θ_r_actual| ≤ 0.05°  且  |θ_p_actual| ≤ 0.05°

其中：
  θ_r_actual = θ_r_track - θ_r_platform（轨道横滚角 - 平台横滚角）
  θ_p_actual = θ_p_track - θ_p_platform（轨道俯仰角 - 平台俯仰角）
```

## 5.4 ±0.05°精度的物理约束分析

### 5.4.1 测距分辨率要求

```
轨道倾角 θ_r = arcsin((D_B - D_A) / L)

对 θ_r 求微分：
  dθ_r = d(D_B - D_A) / (L × cos(θ_r))
  
在 θ_r ≈ 0° 时：
  dθ_r ≈ d(ΔD) / L

要满足 θ_r 精度 ≤ 0.05°：
  d(ΔD) ≤ 0.05° × L × π/180
       ≤ 0.000873 × 300
       ≤ 0.26 mm

即：两侧测距差值分辨率需 ≤ 0.26 mm
```

### 5.4.2 传感器选型要求

| 参数 | 要求 | 说明 |
|---|---|---|
| 测距精度 | ±0.15 mm | 现场综合精度 |
| 差值分辨率 | ≤ 0.26 mm | 保证±0.05°调姿精度 |
| 采样频率 | ≥ 1000 Hz | 2000Hz更佳 |
| 量程 | 50~300 mm | 安装高度匹配 |

### 5.4.3 逆运动学求解精度

```
ΔL_i = L_target - L_current

±0.05°姿态角对应的缸长变化量：
  ΔL ≈ r_p × sin(0.05°) ≈ 100mm × 0.000873 ≈ 0.087mm

电动缸分辨率需 ≤ 0.05mm → 选用0.01mm分辨率位移传感器
```

## 5.5 实时性保障

### 5.5.1 多线程架构

```
主线程（Jetson AGX Orin）
  ├── 数据采集线程（2000Hz）→ 传感器数据读取
  ├── 融合算法线程（200Hz）→ EKF状态估计
  ├── 控制算法线程（1000Hz）→ 逆运动学 + PID
  └── 通信线程（1000Hz）→ EtherCAT指令下发

实时线程（EtherCAT控制器）
  └── 伺服控制周期（1ms）→ 电机驱动
```

### 5.5.2 缓存与共享内存

- 传感器数据通过循环缓存传递，避免阻塞
- 融合结果通过共享内存与控制线程交换
- 控制指令通过EtherCAT环形缓冲发送

| 01 | 轨道高程实时检测 | 单点测距传感器A/B | 高程序列D_A, D_B |

| 04 | 逆运动学求解 | Stewart平台运动学 | 6缸目标长度 |
| 05 | PID伺服控制 | 6轴伺服驱动器 | 电动缸伸缩指令 |
| 06 | 姿态实时补偿 | Stewart平台执行 | 平台恢复水平 |
| 07 | 远程监控 | 4G/5G通信 | 实时姿态数据上传 |
| 08 | 故障自恢复 | 系统自检 | 安全姿态复位 |
