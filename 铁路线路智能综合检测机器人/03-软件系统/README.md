# 03-软件系统

> **定位说明：** 软件系统是智能综合检测机器人的智慧中枢，将传感器采集的原始数据转化为可用的检测结论。本目录所有软件架构、代码、接口文档均围绕"实时采集→实时处理→实时上传→实时判级"这一核心处理链条展开，服务于综合检测的闭环落地。

## 二、核心软件模块

### 2.1 采集控制模块（SCM - Sensor Control & Management）

负责所有传感器的采集调度，是综合检测数据入口。

**核心功能：**

| 功能 | 说明 |
|-----|------|
| 传感器统一初始化 | 上电后自动识别所有相机/激光/陀螺仪，建立连接 |
| 触发同步控制 | 按帧率配置表（CAM1~CAM6定稿参数）统一触发采集 |
| Frame ID管理 | 每帧数据打唯一 Frame ID，关联里程脉冲 |
| 丢帧监测 | 实时统计每路相机丢帧率、缺失率、异常率 |
| 异常恢复 | 单路传感器异常时自动重连，不影响其他路采集 |

**采集流程：**

```
传感器上电 → 自动识别 → 参数配置 → 软触发/里程触发 → Frame ID打标 → 数据入库
     ↑
  异常监测循环 → 自动重连（单路故障不影响全局）
```

### 2.2 AI缺陷检测模块（IDD - Intelligent Defect Detection）

将原始图像数据实时转化为缺陷检测结论，是智能化的核心。

**算法能力：**

| 能力 | 说明 |
|-----|------|
| 缺陷类型识别 | 裂纹/掉块/波磨/凹陷/腐蚀等（可扩展） |
| 像素级定位 | 输出缺陷在图像中的像素坐标 |
| 置信度评分 | 每项检测输出置信度，过低则标记为疑似 |
| 缺陷分类 | 按类型+形态自动分类 |
| 实时推理 | 边缘端实时推理，不依赖云端 |

**模型框架：**

| 模块 | 技术选型 |
|-----|---------|
| 目标检测网络 | YOLOv8（轻量化+高推理速度） |
| 语义分割网络 | ResNet + UNet（像素级掩码） |
| 模型格式 | ONNX Runtime（跨平台推理，支持边缘部署） |
| GPU加速 | CUDA + TensorRT（INT8量化加速） |

### 2.3 几何参数计算模块（GPC - Geometric Parameter Calculation）

将3D激光点云和陀螺仪数据转化为可量化的几何参数。

**计算内容：**

| 检测项目 | 输入数据 | 输出 |
|---------|---------|------|
| 轨头廓形 | 3D线激光点云 | 廓形偏差、磨耗量、横移量 |
| 轨距 | 3D激光/测距传感器融合 | 轨距值、超限里程 |
| 水平 | 双陀螺仪 | 水平偏差值 |
| 高低 | 双陀螺仪 | 高低不平顺值 |
| 轨向 | 双陀螺仪 | 轨向偏差值 |

**计算原则：**
- 所有几何参数计算结果带里程标签和时间戳
- 计算值与《铁路线路检测标准规范》中的限值实时比对
- 超限值自动触发报警

### 2.4 判级引擎（GE - Grading Engine）

将检测结果自动映射为等级结论，是"综合"二字落地的关键。

**判级标准体系（对接国标）：**

| 等级 | 含义 | 处理动作 |
|-----|------|---------|
| 正常 | 偏差在允许范围内 | 正常记录入库 |
| 一级（轻微） | 存在轻微异常，接近阈值 | 标记关注，安排复检 |
| 二级（中等） | 明显超限，需计划养护 | 列入养护计划 |
| 三级（严重） | 严重影响行车安全 | 立即报警，优先处置 |

**多维判级逻辑：**

```
单维判级（各检测项独立）
      │
      ▼
多维融合（2D缺陷 + 3D廓形 + 姿态参数联合）
      │
      ▼
综合判级结论（考虑缺陷类型+位置+趋势+历史数据）
```

> **综合检测的独特价值：** 单一维度可能是"疑似"，多维数据印证后判级更准确。例如，2D图像检出"疑似裂纹"，3D廓形显示"同位置有凹陷"——两种方法同时指向同一结论，判级置信度大幅提升。

### 2.5 实时报警模块（RAM - Real-time Alarm Module）

三级缺陷的发现与上报必须在现场完成。

**报警机制：**

| 报警等级 | 触发条件 | 响应方式 |
|---------|---------|---------|
| P0（紧急） | 三级缺陷检出 / 急停触发 | 现场声光报警 + 云端实时推送 + 立即停检 |
| P1（重要） | 二级缺陷检出 / 蓄电池欠压 | 云端推送 + 本地记录 + 计划养护 |
| P2（提示） | 一级缺陷 / 数据异常 | 本地记录 + 定期汇总报告 |

**报警链路：**

```
检测结论 → 判级 → P0触发 → 现场声光 → 4G/5G上传 → 云端/手机推送
                         │
                         ▼
                    工控机记录日志
```

### 2.6 数据上传与同步模块（DUS - Data Upload & Sync）

综合检测数据量大，上传策略必须精细化管理。

**分级上传策略（与硬件设计一致）：**

| 级别 | 数据内容 | 上传方式 | 实时性要求 |
|-----|---------|---------|----------|
| P0 | 急停事件、关键告警 | 立即发送 | 毫秒级 |
| P1 | 核心几何结果、异常摘要 | 实时上传 | 秒级 |
| P2 | 压缩后的关键图像/轮廓片段 | 准实时 | 分钟级 |
| P3 | 大体量原始数据（图像/点云） | 批量延时上传 | 小时级/缓存后批量 |

**断点续传机制：**

- 网络中断时，原始数据本地SSD缓存
- 网络恢复后自动从断点位置继续上传
- 上传状态在工控机本地数据库记录，可查询可追溯

## 五、软件系统设计原则

**1. 实时性优先**
- 边缘计算链路延迟控制在秒级以内
- 关键告警（P0）处理链路延迟 < 500ms

**2. 可靠性为本**
- 单模块故障不影响其他模块运行
- 关键数据双重存储（内存缓存+SSD持久化）

**3. 可维护性**
- 参数配置与代码逻辑分离，参数可在触摸屏上直接调整
- 所有模块具备日志输出，支持现场故障诊断

**4. 可扩展性**
- 新增缺陷类型只需扩展AI模型，无需改动架构
- 新增检测项目只需添加传感器配置，框架不变
