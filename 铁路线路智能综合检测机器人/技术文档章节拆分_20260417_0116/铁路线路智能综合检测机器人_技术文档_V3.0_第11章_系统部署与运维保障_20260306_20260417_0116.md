## 11 系统部署与运维保障

### 一、5 步现场部署

本系统支持快速部署，单人可在 30 分钟内完成从设备到场到开始检测的全部准备工作。

**部署流程总览：**

```
设备到场 → 设备就位 → 开机自检 → 参数确认 → 一键出发
（5分钟）  （10分钟）  （5分钟）   （5分钟）   （即开始）
```

**详细部署步骤：**

| 步骤 | 操作 | 时长 | 详细说明 |
|---|---|---|---|
| **1. 设备就位** | 将检测机器人搬运至铁轨起点 | 5min | 两人抬运至轨道上，4轮自动对准轨道中心线 |
| **2. 开机自检** | 按下电源开关，系统自动启动 | 5min | 传感器自检 / 相机曝光校正 / 单点测距传感器功率校准 / HWT905 IMU零偏校准，触摸屏显示各设备状态 |
| **3. 参数确认** | 触摸屏输入本次任务参数 | 5min | 检测里程区段（起始/终点）、检测速度（默认1m/s）、任务名称 |
| **4. 一键出发** | 按下"开始任务"，机器人启动 | 即开始 | 自动定速巡航至目标速度，全程无需人工干预 |
| **5. 任务结束** | 自动返航或手动"一键返航" | 即触发 | 到达终点自动减速停车，或手动一键返航；数据自动打包上传云端 |

**部署就绪检查清单（出发前必查）：**

| 检查项 | 标准 | 异常处理 |
|---|---|---|
| 工业相机×6 | 全部在线，心跳正常 | 离线相机标记，冗余相机自动接管 |
| 单点测距传感器矩阵 | 全部在线，数据率稳定，测量精度±1mm | 单传感器降级运行，标记数据缺失区段，任务继续 |
| HWT905 IMU | 数据一致，姿态正常，零偏稳定 | 零偏超限自动补偿，异常时触发自检，人工检修 |
| **测距传感器矩阵** | **全部在线，数据率稳定，测量精度±1mm** | **标记该传感器离线，启动冗余逻辑，任务后检修** |
| BMS电量 | SOC ≥ 80%（或任务所需电量） | 电量不足先充电或更换电池 |
| 存储空间 | 磁盘剩余 ≥ 20% | 删除历史数据后继续 |
| 网络连接 | 信号强度 > -90dBm | 切换为本地存储模式，任务后补传 |
| 定位信号 | GPS/北斗定位正常 | 使用里程计推算，标记无定位区段 |

> 出发前检查可在触摸屏上一键执行，系统自动逐项检测并报告结果，全部通过方可出发。

### 二、运维保障体系

本系统建立了覆盖设备全生命周期的运维保障体系，确保系统长期稳定运行。

**运维保障体系架构：**

```
┌─────────────────────────────────────────┐
│           运维保障体系                      │
│                                          │
│  远程运维 ──→ 定期标定 ──→ 备件支持      │
│       │              │              │    │
│       ↓              ↓              ↓    │
│  故障诊断        传感器校准       快速更换    │
│  参数调优        精度验证        供应链保障  │
│  固件升级                           │    │
│       │                             ↓    │
│       └────→ 软件维护 ←←←←←←←←←←←←┘    │
│                                          │
│  数据维护 ──→ 定期备份 ──→ 多副本存储     │
└─────────────────────────────────────────┘
```

**远程运维详解：**

| 运维内容 | 说明 | 方式 |
|---|---|---|
| 故障诊断 | 远程登录查看实时状态、日志、告警记录，定位故障根因 | 4G/5G网络接入 |
| 参数调优 | 远程调整相机曝光/帧率/检测阈值，无需到场 | 云端下发参数包 |
| 固件升级 | 传感器固件（相机/单点测距传感器/HWT905）远程推送升级 | OTA升级 |
| AI模型更新 | 新增病害类型/优化识别精度，远程推送模型更新包 | 云端模型下发 |
| 系统重启 | 远程重启工控机或重置单节点服务 | 云端指令下发 |

**定期标定制度：**

| 标定项目 | 周期 | 方法 | 工具 |
|---|---|---|---|
| 相机内参标定 | 每季度或更换镜头后 | 张正友标定法，采集20+张标定板图像 | 9×6棋盘格标定板 |
| 相机外参标定 | 每季度或碰撞后 | 基于轨面特征点，三角测量优化外参 | 轨面特征标定件 |
| 单点测距传感器标定 | 每季度或测量精度超差后 | 标准距离参考物，测量并校正测距偏差 | 标准距离标定尺 |
| HWT905 IMU零偏校准 | 每次任务前（自动） | 静止状态下采集零偏值并自动补偿 | 内置自动校准 |
| 测距传感器矩阵标定 | 每季度或测量精度超差后 | 标准距离参考物对比测量，校正测距偏差 | 标准距离标定尺 |
| 里程计标定 | 每季度或轮径磨损后 | 已知距离轨道段，校正轮径参数 | 标准里程标定段 |

**备件支持体系：**

| 备件类型 | 规格 | 数量建议 | 更换时长 |
|---|---|---|---|
| 工业相机 | 同型号同规格 | 2~4台 | < 30min（模块化插拔） |
| 单点测距传感器 | 同型号同规格 | 1~2台 | < 30min |
| 蓄电池 | 同规格同容量 | 1~2组 | < 20min |
| HWT905 IMU | 同型号 | 1台 | < 30min |
| 测距传感器矩阵 | 同型号同规格 | 1~2套 | < 30min |
| 伺服运控系统 | 同型号同规格 | 1套 | < 1h |
| 工控机 | 同配置 | 1台 | < 1h（系统恢复） |

**软件维护：**

| 维护内容 | 周期 | 说明 |
|---|---|---|
| AI模型迭代 | 按需，季度更新 | 根据新病害样本持续优化识别精度 |
| 固件更新 | 按需，月度检查 | 修复已知问题，优化性能 |
| 系统补丁 | 按需 | 安全补丁、功能优化 |
| 配置备份 | 每周自动 | 配置文件自动备份至云端 |

**数据维护：**

| 维护内容 | 周期 | 说明 |
|---|---|---|
| 云端备份 | 实时+每日增量 | 检测数据实时回传，云端实时备份；每日增量备份 |
| 本地备份 | 实时 | SSD本地存储，网络中断时不丢失数据 |
| 多副本存储 | 每日 | 数据副本分布在不同存储节点，防单点故障 |
| 历史归档 | 季度 | 超过1年的历史数据归档至冷存储，降低成本 |

### 三、故障恢复机制

本系统针对各类故障场景设计了分级处理机制，确保故障发生时系统可快速恢复，最大限度减少对检测任务的影响。

**故障分级处理体系：**

| 故障等级 | 定义 | 处理策略 | 影响 |
|---|---|---|---|
| **L1 微故障** | 单传感器降级，不影响核心功能 | 系统自动降级运行，标记数据缺失区段，继续任务 | 任务不中断，数据部分降质 |
| **L2 一般故障** | 设备离线但冗余覆盖 | 冗余接管，任务继续，事后运维处理 | 任务继续，数据有缺失 |
| **L3 严重故障** | 核心设备故障，需立即处理 | 触发返航，数据已采集部分保留 | 任务中断，返航后处理 |
| **L4 紧急故障** | 硬件保护/急停触发 | 立即停车，硬件保护，数据安全保存 | 任务中断，等待人工介入 |

**典型故障处理详情：**

| 故障类型 | 等级 | 处理机制 | 恢复方式 |
|---|---|---|---|
| 单相机离线（6路中1路） | L1 | 标记该相机数据缺失，其余5路覆盖检测，任务不中断 | 任务后更换相机，不影响本次任务 |
| 单点测距传感器离线 | L2 | 保留2D检测功能，降级运行，标记测距数据缺失区段 | 及时维修，任务后更换 |
| 单点测距传感器全部离线（2路均离线） | L3 | 触发返航，已采集数据完整保存 | 返回后检修 |
| HWT905 IMU单轴数据异常 | L1 | 自动切换冗余数据，校验后继续，异常轴标记 | 任务后检修更换 |
| HWT905 IMU双轴同时异常 | L3 | 触发返航，IMU数据异常不影响已采集数据保存 | 返回后检修 |
| 测距传感器矩阵异常 | L1 | 标记该传感器数据缺失，启动冗余逻辑，任务继续 | 任务后更换或重校正 |
| 测距传感器全部离线 | L3 | 触发返航，已采集数据完整保存 | 返回后检修，更换传感器 |
| 网络中断 | L1 | 本地SSD持续存储，网络恢复后自动续传，任务继续 | 自动恢复，无需人工干预 |
| 工控机死机 | L2 | 硬件看门狗检测，3min内自动重启，任务从断点续跑 | 自动恢复 |
| 电量耗尽（< 5%） | L3 | 自动触发定点返航，已采集数据全部保存上传 | 更换电池后继续任务 |
| 工业总线通信中断 | L4 | 硬件急停立即触发，机器人停止，保护设备和线路安全 | 人工检查总线连接 |
| 磁盘空间不足 | L1 | 自动删除最旧任务数据，优先保证当前任务执行 | 任务后清理归档 |
| GPS/定位信号丢失 | L1 | 切换至里程计推算，标记无定位区段，任务继续 | 信号恢复后自动切换回定位 |

**故障恢复流程：**

```
故障发生
    ↓
系统自动识别故障等级（L1~L4）
    ↓
L1/L2 → 自动降级/冗余接管，任务继续
L3 → 自动触发返航，数据保存
L4 → 硬件保护停车，等待人工介入
    ↓
运维人员收到告警（短信/邮件/APP推送）
    ↓
远程诊断 → 判断故障类型
    ↓
远程可解决 → 远程处理（调参/重启/升级）
远程无法解决 → 派发检修工单，安排现场出勤
    ↓
故障修复，系统恢复正常，记录归档
```

**关键设计原则：**

| 原则 | 说明 |
|---|---|
| 数据优先 | 任何故障情况下，已采集数据必须完整保存，不允许因故障丢失数据 |
| 冗余保护 | 核心检测功能均有冗余设计，单点故障不导致任务完全中断 |
| 自动恢复 | 软件类故障优先自动恢复，减少人工干预时间 |
| 安全第一 | 硬件/总线类故障优先保护设备安全，触发硬保护停车 |
| 可追溯 | 所有故障均记录完整日志，便于事后分析和改进 |
