# Rail Sync Diagnostics

## 诊断目标
验证多源数据是否满足“同帧号、同里程基准、可追溯”的一致性要求。

## 输入数据
- 图像帧索引（CAM1~CAM6）
- 3D线激光轮廓帧
- 测距传感器采样记录
- 姿态传感器采样记录
- 统一 Frame ID 与里程脉冲日志

## 核心检查项
1. Frame ID 连续性（是否跳变、重复、倒序）
2. 里程脉冲对齐（同一里程段多源是否落在同一Frame ID窗口）
3. 时间戳漂移（多源时间差是否超阈值）
4. 丢帧率、缺失率、异常率统计
5. 告警可追溯性（告警是否能回溯到原始数据）

## 建议阈值（默认）
- Frame ID连续性错误率：接近0
- 丢帧率：< 0.1%（可按工况调整）
- 多源时间对齐误差：按系统时钟精度与采样周期设定

## 输出格式
- 同步一致性结论：通过 / 预警 / 报警
- 问题列表：问题类型、发生区间、影响源
- 修复建议：参数调整、触发策略优化、网络/缓存优化
- 复测建议：复测时长、速度工况、验收阈值

## 排障优先级
P0: Frame ID错位、严重失同步
P1: 间歇性丢帧、时间漂移超阈值
P2: 统计指标轻微劣化但可运行
