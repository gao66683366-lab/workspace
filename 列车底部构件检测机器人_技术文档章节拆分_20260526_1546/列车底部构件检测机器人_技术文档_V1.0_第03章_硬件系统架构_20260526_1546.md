# 第3章 硬件系统架构

## 3.1 总体硬件架构

本系统采用**分布式模块化**硬件架构，所有模块支持热插拔，便于现场快速维护。系统由六大核心硬件模块组成：

```
┌─────────────────────────────────────────────────────────────┐
│                    列车底部构件检测机器人                     │
│                    硬件系统总体架构                            │
├─────────────┬─────────────┬─────────────┬───────────────────┤
│ 感知层      │ 运动层      │ 计算层      │ 交互层             │
│ ─────────  │ ─────────  │ ─────────  │ ─────────        │
│ · 摄像模块   │ · 轨底平台  │ · 主控单元  │ · 手持终端         │
│ · 照明模块   │ · 电动云台  │ · AI加速    │ · 急停装置         │
│ · 传感融合   │ · 线缆收放  │ · 通信单元  │ · 状态指示         │
└─────────────┴─────────────┴─────────────┴───────────────────┘
```

**六大核心模块：**

| 序号 | 模块名称 | 核心功能 |
|---|---|---|
| 1 | 铁路轨道轨底运动平台 | 4轨底驱动轮+4枕木驱动轮+1升降电缸，分时驱动切换 |
| 2 | 云台摄像模块 | 双云台+双相机（仰拍底架+轨道监控） |
| 3 | 照明系统 | 补光、频闪抑制、去油污反光 |
| 4 | 传感融合单元 | 激光雷达（车底3D轮廓）、IMU、里程计、碰撞传感 |
| 5 | 主控计算单元 | Intel i7工控机（iGPU做AI推理） |
| 6 | 通信与供电单元 | WiFi6/5G无线、锂电池、电源管理 |

## 3.2 铁路轨道轨底运动平台

### 3.2.1 功能定位

铁路轨道轨底运动平台是本机器人的核心运动部件，位于两根钢轨的轨底间隙内，通过**轨底行走结构**使驱动轮与钢轨轨头下沿接触，实现沿轨底的稳定行走。

### 3.2.2 整体结构

**轮组布局：**

| 轮组类型 | 数量 | 安装位置 | 作用 |
|---|---|---|
| 轨底驱动轮 | 4个 | 轨底位置 | 沿轨道行走驱动（电缸收回时着轨） |
| 枕木驱动轮 | 4个 | 枕木上方 | 沿枕木行走驱动（电缸伸出时着枕木） |
| 升降电缸 | 1个 | 车体中央 | 切换驱动模式+避夹板抬升 |
| 测距传感器 | 1–2个 | 轨底驱动轮前方 | 检测钢轨夹板，提前触发模式切换 |

**工作原理：**
```
【枕木模式】电缸伸出 → 枕木驱动轮着枕木（主动驱动）+ 轨底驱动轮悬空
【轨底模式】电缸收回 → 轨底驱动轮着轨面（主动驱动）+ 枕木驱动轮悬空
```

**驱动切换逻辑：**
- 机器人从一个股道转移到相邻股道时，需切换枕木/轨底模式
- 电缸伸出 → 枕木轮驱动 → 机器人沿枕木移动
- 电缸收回 → 轨底轮驱动 → 机器人进入轨底开始检测
- 两组驱动轮由同一组步进驱动器输出，通过离合机构或电缸切换实现互锁

**车体结构：**

| 项目 | 参数 |
|---|---|
| 结构材质 | 高强度铝合金 |
| 总重量 | <=120kg（含电池） |
| 轨距适应 | 标准轨距1435mm |
| 防护等级 | IP67 |

### 3.2.3 行走驱动系统

| 项目 | 参数 |
|---|---|
| 驱动轮数量 | **8个（轨底驱动轮×4 + 枕木驱动轮×4）** |
| 驱动电机 | 步进电机 ×8 或分时复用驱动器 |
| 驱动器 | 步进驱动器（EtherCAT接口） |
| 控制总线 | EtherCAT（CANopen over EtherCAT） |
| 行走速度 | 0.5 m/s（精细检测模式） |
| 定位精度 | ±20 mm（里程计修正） |

**步进电机选型说明：**
- 0.5 m/s 低速运行，步进电机输出稳定，低速不易丢步
- EtherCAT总线控制，配合闭环控制（ encoder反馈）可实现高精度定位
- 配合里程计实时位置修正，定位精度仍可达±20mm
- 成本低于伺服方案，适合本应用场景

**推荐品牌（闭环步进）：**
- 雷赛 DM系列闭环步进
- 汇川 IS620P闭环步进
- 步科 FC3闭环步进

**步进电机（闭环）说明：** 选用带编码器反馈的闭环步进电机，兼具步进的成本优势和伺服的精度，避免丢步。

### 3.2.4 升降电缸

| 项目 | 参数 |
|---|---|
| 类型 | 电动缸（伺服电缸 或 步进电缸） |
| 数量 | 1个（中央布置） |
| 行程 | 满足轨底驱动轮接触轨面的抬升高度（约50–80mm） |
| 推力 | 满足整车重量<=120kg的抬升需求 |
| 控制方式 | EtherCAT（CANopen over EtherCAT） |
| 定位精度 | ±0.5mm |
| 行程控制 | **支持任意位置停留**（非简单的全伸/全缩，可抬升避夹板） |
| 防护等级 | IP67 |

**电缸控制逻辑：**
- **枕木模式**：电缸伸出，枕木驱动轮着枕木（主动驱动），轨底驱动轮悬空
- **轨底模式**：电缸收回，轨底驱动轮着轨面（主动驱动），枕木驱动轮悬空
- **避夹板模式**：测距传感器检测到钢轨夹板 → 电缸主动抬升/收回轨底驱动轮 → 夹板通过后 → 继续收回轨底驱动轮着轨
- **夹板区切换**：夹板区段内自动切换为枕木驱动轮驱动 → 通过后恢复轨底驱动轮

### 3.2.5 控制架构（完整版）

```
[主控计算机]
  ├── EtherCAT ── [步进驱动] ── [轨底驱动轮电机 ×4]
  │                 └─ [枕木驱动轮电机 ×4]
  ├──        └─ [前云台高精度数字舵机]
  ├──        └─ [后云台标准舵机]
  └──        └─ [电缸驱动器] ── [升降电缸]
```

### 3.2.6 运动控制参数

| 参数 | 目标值 |
|---|---|
| 行走速度 | 0.5 m/s（精细检测模式） |
| 定位精度 | ±20 mm（里程计修正） |
| 前云台控制精度 | <0.1°（高精度数字舵机） |
| 后云台控制精度 | <0.5°（标准舵机） |
| 抬升电缸行程 | 约50–80mm |

### 3.2.7 安全保护机制

| 保护类型 | 实现方式 |
|---|---|
| 碰撞保护 | 前置碰撞传感器，碰撞时立即断电停车 |
| 倾角保护 | 内置倾角传感器，倾斜超限自动停机 |
| 断电保护 | 超级电容应急制动，防止溜车 |
| 急停冗余 | 本地+无线+远程三重急停 |
| **升降互锁** | **电缸未完全收回状态下禁止行走；行走状态下电缸锁止** |
| **钢轨夹板避障** | **测距传感器检测到钢轨夹板时，轨底驱动轮自动抬升或收回，避开夹板后切换枕木驱动轮运行** |
| **模式切换保护** | **两组驱动轮不得同时着轨/着枕木，切换过程有互锁** |

## 3.3 云台摄像模块

### 3.3.1 功能定位

云台摄像模块是系统的"眼睛"。本系统采用**双云台双相机方案**：

| 云台 | 安装位置 | 朝向 | 主要功能 |
|---|---|---|
| **前云台+前相机** | 车体前方 | **仰拍（斜向上45°–90°）** | 专门拍摄车底底架结构（构架、横梁、减震装置、液压元件等） |
| **后云台+后相机** | 车体后方 | 斜向后上方 | 监控轨道行走安全，辅助避障 |

**仰拍设计说明：**
- 车底净空仅0.5–1.2m，机器人处于车底最低位置
- 前云台必须能够大角度仰视（垂直方向+90°可达垂直向上），才能完整拍摄底架结构
- 前相机镜头焦距选择需在0.5–1.2m距离内覆盖车底架宽度（约1.0–1.5m），推荐广角或超广角镜头

### 3.3.2 前相机（底架检测专用）

| 项目 | 参数 | 说明 |
|---|---|---|
| 分辨率 | 4K（3840×2160） | 逐行扫描，无压缩 |
| 传感器类型 | CMOS（全局快门） | 消除运动模糊 |
| 帧率 | >=30fps | 支持行走中实时采集 |
| 动态范围 | >=120dB HDR | 适应车底明暗交替（大灯直射vs暗区） |
| 最低照度 | 0.01 lux（彩色） | 极低照度环境成像 |
| 接口类型 | GigE Vision | 缆线长，布线灵活 |
| 防护等级 | IP67 | 镜头部分强调密封 |

**镜头选型（前相机专用）：**

| 项目 | 参数 | 说明 |
|---|---|---|
| 焦距 | 6–12mm | 广角，适配0.5–1.2m近距离、大视场 |
| 类型 | 自动光圈镜头（DC驱动） | 低光环境自动开大光圈 |
| 光圈 | F1.4–F16 自动 | 大光圈保证低照度成像 |
| 对焦 | 自动对焦（AF） | 距离变化时快速重新对焦 |
| 接口 | C/CS接口 | 主流工业镜头接口 |
| 过滤镜 | 镀膜玻璃+偏振滤光片 | 去油污镜面反光（车轮/金属油污反射强烈） |

**推荐品牌（暂定）：**
- 海康机器人 MV-CH2440（高分辨率，适合细节检测）
- 大华 DH-TPT-4K（4K/30fps，性价比）
- FLIR Oryx 4K（工业级，宽温，-20°C适用）

### 3.3.3 前云台（底架检测云台）

| 项目 | 参数 | 说明 |
|---|---|---|
| 水平转动 | 360° 连续 | 无死角环绕 |
| 垂直转动（俯仰） | **-45° ~ +90°** | **+90°为垂直向上，实现仰拍** |
| 控制精度 | <0.1° | **高精度数字舵机/云台电机（闭环）** |
| 控制响应 | <=50ms | 快速响应 |
| 最大转速 | 水平120°/s，垂直90°/s | 高速运动场景 |
| 驱动方式 | **高精度数字舵机（云台电机）** | 闭环控制，精度<0.1° |
| 重复定位精度 | ±0.05° | 多次回到同一点 |
| 防护等级 | IP67 | 防尘防水防油污 |
| 负载能力 | <=3 kg | 前相机+镜头重量较轻 |

**选型说明：** 前云台承担仰拍底架的核心任务，精度要求<0.1°，需选用**无刷云台电机（Brushless Gimbal Motor）** 或**高精度数字舵机**，普通RC舵机无法满足此精度要求。

**仰拍控制策略：**
- 预设位控制：预设车底底架仰拍基准位（+60°~+75°），一键切换至仰拍模式
- 手动微调：操作员实时精细调整俯仰角度
- 自动追踪：结合里程计，前云台随行走位置自动切换仰拍角度

## 3.6 主控与边缘计算单元

### 3.6.1 功能定位

主控单元是系统的大脑，负责运动控制、数据采集、AI缺陷推理、通信管理。AI推理直接运行在主控计算机上（利用Intel i7集成GPU/iGPU进行推理），无需独立AI加速硬件。

### 3.6.2 主控计算机规格

| 项目 | 推荐规格 |
|---|---|
| 处理器 | Intel Core i7-10710U（6核12线程）或同等工控级 **带集成GPU（iGPU）** |
| 内存 | >=16GB DDR4 |
| 存储 | >=512GB SSD（系统）+ >=1TB SSD（数据） |
| 操作系统 | Linux / Windows IoT |
| 通信接口 | GigE×4，USB3.0×2，RS485×2，CAN×1，HDMI×1 |
| 供电 | 24V DC，功耗约40W（含AI推理） |
| 防护等级 | IP54 |
| 工作温度 | -20°C ~ +60°C |
| 散热方式 | 无风扇被动散热 |

**AI推理说明：**
- Intel i7集成GPU（Iris Xe/UHD）可支持INT8量化推理
- 推理框架：ONNX Runtime + OpenVINO（Intel优化）
- 2路1080P实时AI推理可行（降低分辨率节省算力）；如需4K，需优化模型或降低帧率

**备选方案：**
- 研华 UNO-2484G（工业级，Intel Core i7，无风扇）
- 凌华 MXE-5500（工业级，宽温）

### 3.6.3 主控架构（简化版，无独立AI加速单元）

```
[工业相机 GigE] ──→ [主控计算机]
                        ├── [AI推理（iGPU）] → [缺陷结果]
                        ├── [视频编码存储] → [本地SSD]
                        └── [运动控制] ←→ [EtherCAT] → [步进驱动]
```

**简化说明：** 取消独立AI加速单元（Jetson Orin NX），AI推理直接在主控计算机iGPU上运行，系统更简洁、成本更低、功耗更低。

## 3.7 通信与供电单元

### 3.7.1 通信方案：三网物理隔离

本系统通信架构采用**三网物理隔离**设计，将控制网络、采集网络、传输网络完全独立部署，满足工业控制实时性与数据安全要求。

| 通信类型 | 规格 | 用途 | 优先级 |
|---|---|---|---|
| **EtherCAT 控制网** | 100Mbps，硬实时<=1ms | **控制网络**：步进驱动、运动控制、急停 | **主用** |
| **千兆以太网（采集网）** | GigE Vision，>=1Gbps | **采集网络**：工业相机数据、激光雷达点云 | **主用** |
| **5G/WiFi6/光纤（传输网）** | 5G NR / WiFi6 / 光纤 | **传输网络**：远程监控、检测数据上传 | **主用** |

**三网物理隔离说明：**
- 控制网络：EtherCAT硬实时总线，毫秒级响应，驱动步进电机运动
- 采集网络：千兆以太网，专用于相机/激光雷达高速数据采集
- 传输网络：5G/WiFi6/光纤，检测数据远程上传、远程监控
- 三网完全独立，任一网络故障不波及其他网络

### 3.7.2 供电系统配置

| 项目 | 参数 |
|---|---|
| 电池类型 | 磷酸铁锂电池组（LiFePO4） |
| 电池电压 | 48V DC |
| 电池容量 | **16–20Ah（续航约2h @ 标准作业）** |
| 充电方式 | 离线充电（取出电池）或 线充（DC接口） |
| 充电时间 | <=2h（0–80%） |
| 电源管理 | BMS（电池管理系统），过充/过放/过温保护 |
| 应急供电 | 超级电容（用于急停执行和断电保护） |
| 系统总功耗 | **<=120W（典型）/ <=200W（峰值）** |

**功耗预算：**

| 模块 | 典型功耗 | 峰值功耗 |
|---|---|---|
| 主控计算机（含AI推理） | 40W | 55W |
| 云台（×2）+ 相机（×2） | 25W | 40W |
| 照明系统 | 15W | 40W（瞬间） |
| 里程计+IMU+激光雷达 | 10W | 15W |
| 通信模块（WiFi） | 5W | 8W |
| **系统总计** | **~95W** | **~158W** |

**续航说明：**
- 电池16Ah × 48V ≈ 768Wh
- 系统典型功耗约95W，理论续航约8h
- 满足单次作业续航>=4h要求
- 考虑电池衰减、低温、照明峰值等综合因素，实际续航>=4h
