# 第3章 主控板配置

## 3.1 概述

主控板是机器人的核心中枢，采用**双系统架构**——运动控制与相机控制物理隔离、独立运行，通过内部总线协同。本章详细描述主控板的硬件架构、接口定义、通信协议和电气环境规格。

## 3.2 双系统架构

### 3.2.1 系统划分

| 系统 | 负责功能 | 核心芯片 | 实时性要求 |
|---|---|---|---|
| **运动控制系统** | 电机驱动、里程计、姿态传感器、测距传感器、激光雷达、云台舵机、BMS监控 | ARM Cortex-R5 / DSP | 硬实时（≤1ms） |
| **相机控制系统** | 前视/后视相机控制、视频编码、WiFi传输、光源控制 | ARM Cortex-A53 + FPGA/NPU | 软实时（≤50ms） |

### 3.2.2 双系统协同机制

```
┌─────────────────────────────────────────────────────────┐
│                      主 控 板 内 部                       │
│                                                         │

│  │   运动控制系统     │ ←──→ │   相机控制系统     │      │
│  │   (实时核)        │ CAN  │   (计算核)        │      │
│  │                   │ 总线 │                   │      │
│  │  · 12轴电机驱动   │      │  · GigE Vision×2  │      │
│  │  · 里程计接口     │      │  · H.264/H.265编码│      │
│  │  · 485传感器接口  │      │  · WiFi 6传输     │      │
│  │  · 激光雷达接口   │      │  · LED光源控制    │      │
│  │  · 舵机云台控制   │      │                   │      │
│  │  · BMS监控        │      │                   │      │
│  └───────────────────┘      └───────────────────┘      │

│            └────────┬────────────┘                       │
```

### 3.2.3 双系统通信协议

**CAN总线消息格式：**

| 字段 | 长度 | 说明 |
|---|---|---|
| CAN ID | 11bit | 消息标识符（优先级） |
| DLC | 4bit | 数据长度（0~8字节） |
| Data | 0~8B | 负载数据 |
| CRC | 15bit | 循环冗余校验 |

**关键CAN消息定义：**

| CAN ID | 发送方 | 接收方 | 周期 | 数据内容 |
|---|---|---|---|---|
| 0x101 | 运动系统 | 相机系统 | 10ms | 当前速度、里程、运行状态 |
| 0x102 | 运动系统 | 相机系统 | 100ms | 姿态角（横滚/俯仰/航向） |
| 0x201 | 相机系统 | 运动系统 | 10ms | 缺陷标记触发信号 |
| 0x202 | 相机系统 | 运动系统 | 100ms | 光源亮度设置 |
| 0x301 | 运动系统 | 双系统 | 1000ms | BMS总数据（电压/电流/SOC） |
| 0x401 | 双系统 | 看门狗 | 100ms | 心跳信号 |

## 3.4 通信协议

### 3.4.1 控制指令协议（TCP）

控制端到主控板的指令通过TCP传输，端口号5000。

**指令帧格式：**

```
+--------+----------+----------+----------+
| Header | Length   | Payload  | Checksum |
| 2字节   | 2字节    | N字节    | 2字节    |
+--------+----------+----------+----------+
```

| 字段 | 值 | 说明 |
|---|---|---|
| Header | 0xAA 0x55 | 帧头标志 |
| Length | N | 后续Payload长度 |
| Payload | JSON格式 | 指令内容 |
| Checksum | CRC16 | 校验和 |

**控制指令示例：**

```json
// 运动控制
{"cmd": 0x01, "subcmd": 0x01, "speed": 0.4, "dir": 1, "ts": 1718000000}

// 参数：cmd=0x01(运动), subcmd=0x01(速度设置), speed=速度值m/s, dir=方向(1=前进/0=停止/-1=后退), ts=时间戳

// 云台控制
{"cmd": 0x02, "subcmd": 0x01, "camera": 0, "pan": 45, "tilt": -30, "ts": 1718000001}

// 参数：cmd=0x02(云台), subcmd=0x01(角度控制), camera=0(前)/1(后), pan=水平角度, tilt=垂直角度

// 光源控制
{"cmd": 0x03, "subcmd": 0x01, "brightness": 80, "ts": 1718000002}

// 参数：cmd=0x03(光源), brightness=0~100(亮度百分比)
```

### 3.4.2 传感器数据协议（UDP）

传感器数据通过UDP回传，端口号5001，周期1Hz。

**姿态传感器数据帧（HWT905）：**

```json
{
  "type": "imu",
  "roll": 0.52,      // 横滚角，度
  "pitch": 1.23,    // 俯仰角，度
  "yaw": 45.67,     // 航向角，度
  "wx": 0.012,      // 角速度X，deg/s
  "wy": 0.023,      // 角速度Y，deg/s
  "wz": 0.034,      // 角速度Z，deg/s
  "ax": 0.021,      // 加速度X，g
  "ay": 0.032,      // 加速度Y，g
  "az": 1.002,      // 加速度Z，g
  "ts": 1718000000
}
```

**测距传感器数据帧：**

```json
{
  "type": "ranging",
  "front": 456.7,   // 前方距离，mm
  "rear": 123.4,    // 后方距离，mm
  "ts": 1718000000
}
```

**里程数据帧：**

```json
{
  "type": "odometry",
  "distance": 12345.6,  // 累计里程，mm
  "speed": 0.42,        // 当前速度，m/s
  "ts": 1718000000
}
```

**BMS数据帧：**

```json
{
  "type": "bms",
  "voltage": 49.2,      // 总电压，V
  "current": -2.3,      // 电流，A（负=放电）
  "soc": 87,            // 荷电状态，%
  "temp_max": 35,       // 最高温度，℃
  "temp_min": 33,       // 最低温度，℃
  "cycle_count": 150,   // 循环次数
  "status": "normal",   // 状态
  "ts": 1718000000
}
```
