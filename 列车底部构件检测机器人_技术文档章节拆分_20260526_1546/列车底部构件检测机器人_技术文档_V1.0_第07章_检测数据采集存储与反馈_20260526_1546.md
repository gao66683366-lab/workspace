# 第7章 检测数据采集、存储与反馈

## 7.1 数据采集概述

本系统数据采集覆盖视频/图像、激光雷达点云、传感器状态三大类，数据同步精度<=10ms，确保各数据源时间对齐。

| 数据类型 | 采样频率 | 说明 |
|---|---|---|
| 视频/图像 | 4K@30fps | 工业相机采集，H.265编码 |
| 激光雷达点云 | 10–15Hz | 障碍物检测与轮廓扫描 |
| IMU姿态数据 | 200Hz | 姿态角、加速度、角速度 |
| 里程计数据 | 100Hz | 行走距离累加 |
| 倾角数据 | 50Hz | 平台倾斜角度 |
| 缺陷AI结果 | 每帧触发 | 实时缺陷标记 |
| 系统状态 | 1Hz | 温度、电流、电压、故障码 |

## 7.2 视频采集系统

### 7.2.1 多相机部署方案

| 相机编号 | 安装位置 | 视角方向 | 主要覆盖区域 |
|---|---|---|---|
| 相机1 | 云台正前方 | 斜向下45° | 轨道正前方区域 |
| 相机2 | 云台侧方 | 水平侧向 | 轮对侧面、轴箱 |
| 相机3 | 云台侧方（对称） | 水平侧向（反向） | 对侧轮对侧面 |
| 相机4 | 车体底部中间 | 垂直向下 | 底架中央区域 |

**说明：** 相机数量和安装位置可根据实际车厢结构调整，标准配置为1台云台相机（可覆盖主要区域），扩展配置支持4路固定相机。

### 7.2.2 视频编码参数

| 项目 | 参数 |
|---|---|
| 编码格式 | H.265（HEVC）Main Profile |
| 分辨率 | 4K（3840×2160）或 1080P（1920×1080） |
| 帧率 | 30fps |
| 码率控制 | CBR，8–20Mbps自适应 |
| GOP长度 | 30帧 |
| 色彩空间 | YUV 4:2:0 |
| 音频 | 可选（环境音记录） |

### 7.2.3 图像水印叠加

每帧图像强制叠加以下水印信息（不可篡改）：

| 水印字段 | 格式 | 示例 |
|---|---|---|
| 时间戳 | YYYY-MM-DD HH:mm:ss.ms | 2026-05-26 15:30:05.123 |
| 行走距离 | 单位：m，保留3位小数 | 12.456 m |
| 定位坐标 | 里程计读数 | KM:12+456 |
| 云台角度 | 水平/垂直角度 | H:45.2° V:-12.8° |
| 车速 | 单位：m/s | V:0.5 m/s |
| 设备编号 | 唯一设备ID | DEV:RB-2026-001 |
| 任务编号 | 本次任务唯一ID | TSK:20260526001 |

## 7.3 数据存储方案

### 7.3.1 存储架构

```
┌─────────────────────────────────────────────────┐
│              本地SSD存储（主存储）               │
│  系统盘：512GB SSD（操作系统+应用）             │
│  数据盘：>=1TB SSD（视频+图像+AI结果）          │
└─────────────────────────────────────────────────┘
                     ↓ 数据同步
┌─────────────────────────────────────────────────┐
│           云端/服务器存储（备份存储）            │
│  关键数据：缺陷图片+报告+元数据                  │
└─────────────────────────────────────────────────┘
```

### 7.3.2 存储容量预算

| 数据类型 | 单车采集量 | 存储格式 | 单车所需空间 |
|---|---|---|---|
| 4K视频流 | 20min×60s×30fps=36000帧 | H.265，约15Mbps | ~3.5 GB |
| 缺陷图片 | ~100张（按平均缺陷数） | JPEG 4K原图 | ~2 GB |
| AI推理结果 | JSON格式元数据 | 文本 | ~10 MB |
| 传感器日志 | 20min全量记录 | 二进制日志 | ~500 MB |
| **单车合计** | | | **~6 GB** |

**存储周期：** >=1TB可用存储 ~= 170辆车的检测数据（循环覆盖）

### 7.3.3 存储管理策略

| 策略 | 说明 |
|---|---|
| 循环覆盖 | 存储满后自动覆盖最早历史数据（默认90天） |
| 重要数据保护 | 缺陷图片+报告标记保护，不被覆盖 |
| 断点续采 | 断电重启后，从断点位置继续采集，不重复采集已完成区域 |
| 数据校验 | MD5校验，防止传输损坏 |

## 7.4 实时数据反馈

### 7.4.1 实时视频流反馈

| 项目 | 说明 |
|---|---|
| 实时预览 | 手持终端实时显示4K视频流，延迟<=200ms |
| 多路同屏 | 支持1/2/4路分屏显示 |
| 画中画 | PIP模式，副画面叠加在主画面角落 |

### 7.4.2 实时缺陷标记反馈

| 项目 | 说明 |
|---|---|
| 标记方式 | 缺陷区域叠加彩色框（红/黄/绿对应严重/一般/轻微） |
| 标记内容 | 缺陷类别标签 + 置信度百分比 |
| 声音提示 | 严重缺陷触发蜂鸣器报警 |
| 界面提示 | 缺陷列表实时更新，可点击跳转查看 |

### 7.4.3 实时统计反馈

| 统计项 | 实时显示 |
|---|---|
| 已检测里程 | 实时累加，单位m |
| 已采集图片数 | 实时累加 |
| 已识别缺陷数 | 按等级分类统计 |
| 任务完成度 | 百分比进度条 |
| 预估剩余时间 | 根据速度估算 |

## 7.5 数据格式规范

### 7.5.1 图像数据格式

| 项目 | 规范 |
|---|---|
| 格式 | JPEG（采图原图）、PNG（缺陷标记图） |
| 分辨率 | 原始4K（3840×2160） |
| 色彩 | sRGB Color Space |
| 元数据 | EXIF + 自定义水印字段 |

### 7.5.2 视频数据格式

| 项目 | 规范 |
|---|---|
| 格式 | MP4（H.265/HEVC） |
| 分辨率 | 3840×2160 或 1920×1080 |
| 帧率 | 30fps |
| 码率 | 8–20Mbps |

### 7.5.3 缺陷记录数据格式（JSON）

```json
{
  "defect_id": "D20260526001",
  "task_id": "TSK20260526001",
  "timestamp": "2026-05-26T15:30:05.123",
  "distance_m": 12.456,
  "position": "走行部_左轮对",
  "defect_type": "crack",
  "severity": 1,
  "confidence": 0.97,
  "image_path": "/data/defect/D20260526001.jpg",
  "bbox": [1200, 800, 1350, 950],
  "device_id": "RB-2026-001"
}
```

### 7.5.4 传感器数据格式（二进制日志）

| 字段 | 字节数 | 说明 |
|---|---|---|
| 时间戳 | 8字节 | 微秒级UTC时间 |
| IMU姿态 | 12字节 | 俯仰/横滚/偏航角 |
| 里程计 | 8字节 | 累计距离（mm） |
| 倾角 | 4字节 | 平台横滚角 |
| 速度 | 4字节 | 实时速度（mm/s） |
| **每帧合计** | **36字节** | |
