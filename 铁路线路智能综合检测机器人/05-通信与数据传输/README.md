# 05-通信与数据传输

> **定位说明：** 通信与数据传输是智能综合检测闭环的命脉——传感器数据采集后，通过可靠的通信链路传输到边缘计算节点，边缘处理结果再通过4G/5G上传至云端。本目录涵盖 EtherCAT 实时控制总线、工业以太网数据传输、4G/5G无线上传三大通信子系统的设计文档。

---

## 一、通信架构总览

```
┌─────────────────────────────────────────────────────────────────────┐
│                   三网物理隔离通信架构                                 │
│                                                                     │
│  ┌─────────────┐     ┌──────────────┐     ┌───────────────────┐    │
│  │ 实时控制网   │     │ 数据采集网   │     │ 外部传输网          │    │
│  │ EtherCAT    │     │ 千兆工业以太网 │    │ 4G/5G工业路由器     │    │
│  │ 硬实时      │     │ 软实时       │     │ 非实时             │    │
│  └──────┬──────┘     └──────┬──────┘     └────────┬──────────┘    │
│         │                   │                     │                 │
│         ▼                   ▼                     ▼                 │
│  伺服驱动器◄──────►工控机◄────────────►工业路由器◄────►云端平台     │
│  （运动指令）     （数据汇聚）        （数据上传）      （云端分析）  │
└─────────────────────────────────────────────────────────────────────┘
```

**三网物理隔离原则：**
- 控制指令（伺服启停/急停）永远不被数据流量堵塞
- 相机/激光数据采集带宽独占，不与控制帧争抢
- 4G/5G上传链路独立，不影响本地实时处理

## 二、EtherCAT 实时控制总线

### 2.1 总线定位

EtherCAT 是本系统的"神经"——负责工控机与伺服驱动器之间的控制指令下发和状态反馈，要求毫秒级硬实时性。

### 2.2 拓扑结构

```
工控机（EtherCAT主站）
      │
      │ EtherCAT网线（菊花链或星型）
      │
      ├──▶ 伺服驱动器1（轮组1）
      ├──▶ 伺服驱动器2（轮组2）
      ├──▶ 伺服驱动器3（轮组3）
      └──▶ 伺服驱动器4（轮组4）
```

### 2.3 通信内容

| 方向 | 内容 | 周期 |
|-----|------|------|
| 主站→从站 | 速度指令、位置指令、使能信号 | 1ms |
| 从站→主站 | 实际速度、实际位置、故障状态 | 1ms |

### 2.4 实时性指标

| 项目 | 指标 |
|-----|------|
| 通信周期 | 1 ms（1000 Hz） |
| 抖动（Jitter） | < 0.1 ms |
| 指令延迟 | < 2 ms（指令发出到电机响应） |
| 协议栈 | IgH EtherCAT Master / SOEM |

## 三、千兆工业以太网数据采集网

### 3.1 总线定位

数据采集网是本系统的"血管"——6个工业相机和2个3D线激光通过 GigE Vision 协议将图像和点云数据汇入工控机，要求大带宽、低丢包。

### 3.2 带宽分配

| 设备 | 规格 | 峰值带宽（估算） |
|-----|------|--------------|
| CAM1~CAM2（轨面） | 2448×2048 @ 20fps RAW | 约 500 Mbps（2路） |
| CAM3~CAM4（螺栓） | 2448×2048 @ 25fps RAW | 约 600 Mbps（2路） |
| CAM5~CAM6（补充） | 2448×2048 @ 20fps RAW | 约 500 Mbps（2路） |
| 3D线激光×2 | 点云数据 @ XX fps | [待确认] |
| **合计** | | **约 1.6~2 Gbps** |

> **网卡选型：** 工控机 PCIe 卡需按峰值带宽的 1.5~2 倍冗余配置，建议单口或双口千兆网卡绑定（Bonding），或采用工业级交换机汇聚后上联。

### 3.3 协议栈

| 层级 | 协议 | 说明 |
|-----|------|------|
| 应用层 | GigE Vision 2.0 | 工业相机标准协议 |
| 传输层 | UDP（图像数据）/ TCP（控制） | 大块数据传输 |
| 网络层 | IPv4 | 工业网络通用 |
| 链路层 | 1000BASE-T | 千兆以太网 |

### 3.4 数据采集策略

- **软触发采集**：工控机发送软件触发指令，相机按配置的帧率持续采集
- **里程触发采集**：结合里程脉冲信号，在特定里程位置触发采集（螺栓区域）
- **Frame ID 同步**：每帧图像带唯一 Frame ID，与里程脉冲绑定，保证多相机同歩

## 四、4G/5G 外部传输网

### 4.1 总线定位

外部传输网是本系统的"出口"——工控机处理结果通过工业路由器上传至云端，铁路现场无固定网络覆盖时，通过蜂窝网络实现数据上云。

### 4.2 传输链路

```
工控机 ──（千兆以太网）──▶ 工业路由器 ──（SIM卡/蜂窝网络）──▶ 云端服务器
                              │
                              ├── 本地Web管理界面
                              └── 支持VPN（数据安全）
```

### 4.3 分级上传机制

| 级别 | 数据内容 | 触发方式 | 实时性 | 备注 |
|-----|---------|---------|-------|------|
| P0 | 急停事件、三级缺陷告警 | 立即触发 | 毫秒级 | 网络中断时本地记录，恢复后补传 |
| P1 | 几何参数结果、异常摘要 | 实时上传 | 秒级 | 最重要结果优先 |
| P2 | 压缩后的关键图像/轮廓片段 | 准实时 | 分钟级 | JPEG压缩后上传 |
| P3 | 原始大体量数据 | 批量上传 | 小时级 | 本地SSD缓存，闲时批量 |

### 4.4 断点续传机制

- 工控机本地记录每条上传记录的发送状态（Ack）
- 网络中断期间数据本地SSD缓存
- 网络恢复后，自动从最后一条 Ack 位置继续上传
- 不丢数据，不重复上传

### 4.5 工业路由器选型要求

| 项目 | 要求 |
|-----|------|
| 蜂窝制式 | 4G LTE Cat.4 以上 / 5G（根据当地网络） |
| 网口 | 至少2个千兆网口 |
| 工作温度 | -30°C ~ +70°C（铁路现场） |
| 防护等级 | IP30（机柜内安装） |
| 供电 | DC 12V/24V 支持（由48V系统DC-DC转换） |
| VPN支持 | IPSec / OpenVPN（数据安全） |
| 管理 | 支持远程管理、Web配置、SNMP监控 |

## 五、通信安全与可靠性

### 5.1 网络安全

| 措施 | 说明 |
|-----|------|
| 网络隔离 | 三网物理隔离，EtherCAT与数据网完全分离 |
| VPN加密 | 4G/5G上传链路全程VPN加密 |
| 防火墙 | 工控机仅开放必要端口，禁止外部主动连接 |
| 认证授权 | 云端API调用需Token认证 |

### 5.2 通信可靠性机制

| 机制 | 作用 |
|-----|------|
| 心跳检测 | 工控机与云端定期心跳，网络中断立即感知 |
| 自动重连 | 网络中断后自动尝试重连，恢复后继续上传 |
| 链路冗余（规划） | 5G+4G双卡绑定，任一网络失效自动切换 |
| 本地缓存 | 中断期间数据本地存，不丢P0/P1级数据 |
