# 第4章 软件与AI算法架构

## 4.1 软件总体架构

本系统软件采用**分层微服务 + 实时多线程**混合架构，高实时、高稳定、易扩展。

```
┌──────────────────────────────────────────────┐
│                 应用层                        │
│  人机交互 │ 任务管理 │ 报告生成 │ 系统对接   │
├──────────────────────────────────────────────┤
│               AI算法层                        │
│  缺陷检测 │ 图像增强 │ 目标识别 │ 语义分割  │
├──────────────────────────────────────────────┤
│              中间件层                         │
│  视频编码 │ 数据采集 │ 定位解算 │ 避障算法   │
├──────────────────────────────────────────────┤
│               系统层                          │
│  实时OS │ 进程调度 │ 异常处理 │ 看门狗      │
├──────────────────────────────────────────────┤
│               驱动层                          │
│  电机驱动 │ 云台驱动 │ 传感器驱动 │ 照明驱动 │
└──────────────────────────────────────────────┘
```

## 4.2 各层功能说明

### 4.2.1 驱动层

| 模块 | 功能说明 |
|---|---|
| 电机驱动 | 左右轮组步进驱动，PWM控制，闭环反馈 |
| 云台驱动 | 水平/垂直伺服控制，支持预设位和手动微调 |
| 传感器驱动 | 采集激光雷达、IMU、里程计、倾角、碰撞信号 |
| 照明驱动 | LED亮度PWM调光，支持频闪同步触发 |
| 通信驱动 | GigE相机驱动、5G/WiFi通信管理 |

### 4.2.2 系统层

| 模块 | 功能说明 |
|---|---|
| 实时操作系统 | Linux + PREEMPT_RT实时补丁，或 RTX实时扩展 |
| 进程调度 | 硬实时任务（运动控制）优先级最高 |
| 异常处理 | 分级异常捕获与恢复（轻微/一般/严重） |
| 看门狗 | 软硬件双看门狗，系统卡死自动复位 |

### 4.2.3 中间件层

| 模块 | 功能说明 |
|---|---|
| 视频编码 | H.265/H.264硬件编码，码流自适应 |
| 数据采集 | 多路GigE相机同步采集，时间戳对齐 |
| 定位解算 | 里程计+IMU融合，实时计算机器人位置 |
| 避障算法 | 激光雷达点云处理，障碍物检测与路径规划 |
| 数据压缩 | 边缘侧数据压缩，减少传输带宽 |

### 4.2.4 AI算法层

| 模块 | 功能说明 |
|---|---|
| 图像增强 | 去噪声、去油污反光、低照度提亮 |
| 目标检测 | 定位缺陷区域（裂纹/锈蚀/渗漏/松动） |
| 缺陷分割 | 像素级缺陷轮廓提取 |
| 图像分类 | 缺陷等级判定（严重/一般/轻微） |

> **说明：** AI算法层详细内容见本章第4.5节。

### 4.2.5 应用层

| 模块 | 功能说明 |
|---|---|
| 人机交互 | 触控界面、摇杆、按键事件响应 |
| 任务管理 | 检测任务创建、调度、执行、归档 |
| 报告生成 | PDF/Excel双格式报告，自动填入缺陷信息 |
| 系统对接 | 与MRO、EAM、工务检修平台数据接口对接 |

## 4.4 边缘计算架构

```
[相机原始帧]
    ↓
[主控计算机] → [H.265视频编码] → [本地存储/实时流]
    ↓
[AI推理（iGPU + OpenVINO INT8）]
    ↓
[缺陷检测推理] → [缺陷标记框+类别+置信度]
    ↓
[结果输出] → [界面实时显示 + 报告归档]
```

**边缘计算策略：**
- 所有AI推理在主控计算机iGPU本地完成（Intel i7集成GPU + OpenVINO优化），无需独立AI加速单元
- 实时性要求<200ms，满足边走边检作业模式
- 支持离线作业，无网络环境下正常检测

## 4.5 AI缺陷识别技术概述

> **说明：** 本节为概述级，AI算法详细设计在后续专项研究中展开。

### 4.5.1 缺陷类型定义

| 缺陷大类 | 具体缺陷 | 典型特征 |
|---|---|---|
| 裂纹 | 轮对裂纹、底架裂纹、焊缝裂纹 | 细线状，低对比度，暗区难辨 |
| 锈蚀 | 金属表面锈蚀、腐蚀穿孔 | 色差，颜色发红/褐 |
| 渗漏 | 油污渗漏、气动泄漏 | 油渍/水渍痕迹，边缘扩散形状 |
| 松动 | 螺栓松动、紧固件脱落 | 位置偏移，角度歪斜 |
| 磨损 | 闸瓦磨耗、打磨轮磨耗 | 厚度减少，摩擦痕迹 |

### 4.5.2 AI算法技术路线（概述）

| 项目 | 技术方向 |
|---|---|
| 检测框架 | 目标检测（YOLO系列）+ 语义分割（Mask R-CNN） |
| 主干网络 | ResNet-50/101 或 EfficientNet（兼顾精度与速度） |
| 推理方式 | **OpenVINO INT8量化推理**，边缘部署（Intel iGPU优化） |
| 缺陷分级 | 置信度阈值 + 形态学后处理 |
| 大机专项 | 基于通用模型的迁移学习，补充大机段专项数据集 |

### 4.5.3 AI性能指标（目标）

| 指标 | 目标值 |
|---|---|
| 缺陷识别准确率 | >=95%（Top-1） |
| 单帧推理时间 | <200ms（iGPU + OpenVINO INT8） |
| 支持缺陷类别数 | >=20类 |
| 误报率 | <=5% |
