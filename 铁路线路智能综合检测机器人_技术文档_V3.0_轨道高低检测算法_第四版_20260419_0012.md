## 第一部分：姿态传感器方案不可行理由

### 1.1 问题定义

铁路轨道高低检测的目标：获取左轨、右轨各自的高程序列，计算左右轨高差，并按波长（短波 <10m / 长波 ≥10m）进行轨道质量评价。

### 1.2 四个核心不可行原因

**原因一：陀螺仪输出角速度，不是位移**

- 陀螺仪输出 °/s（角速度），积分一次得角度，再积分得角位移
- 角位移 ≠ 垂向位移；陀螺仪没有感知沿激光方向绝对位移的能力
- 积分漂移随时间累积，10 秒后误差可达毫米至厘米级

**原因二：车身姿态 ≠ 轨道高程**

姿态传感器装在车体上，输出"车身倾斜了多少度"，无论精度多高，都无法直接回答"左轨比右轨高多少"。车身姿态平稳但左轨下沉时，传感器输出为 0，病害漏检。

**原因三：无法区分左轨和右轨**

- 一套 IMU 装在车体中间，测的是中间那一点的车体姿态
- 1435mm 轨距，左轨、右轨是两个不同的物理位置
- 在一套 IMU 里，左/右轨各自的高程无法分离为两路独立信号

**原因四：跳动干扰无法自消除**

- 车体跳动（1~5Hz）与中波轨道不平顺（λ=1~10m，v=1m/s 对应 0.1~1Hz）频段相邻
- IMU 测到的振动里同时包含"车跳了"和"路不平"，自身无法分离
- 没有独立的空间参考基准，无法将两者剥离

### 1.3 结论

> **纯姿态传感器方案仅能提供车体姿态参考，不能独立完成左右轨独立高程测量。必须引入直接测量轨廓的传感器（3D 线激光）作为主测手段，IMU 仅作为辅助校正。**
> **纯姿态传感器方案仅能提供车体姿态参考，不能独立完成左右轨独立高程测量。必须引入直接测量轨面距离的传感器（单点测距传感器）作为主测手段，HWT905 仅作为辅助校正。**

## 第二部分：完整技术方案（第四版）

### 2.1 传感器配置

| 序号 | 传感器 | 数量 | 角色 | 输出 |
|------|--------|------|------|------|
| 1 | 单点测距传感器 | 2 台 | 主测（高低） | 左右轨面垂向距离 D（测距值） |
| 2 | 3D 线激光传感器 | 2 台 | 廓形检测 | 左右轨轨头廓形点云（X=里程, Y=横, Z=高程） |
| 3 | HWT905 | 1 套 | 辅助校正 | 车体俯仰角 θ_pitch、横滚角 θ_roll |
| 4 | 轮速编码器 | 1 套 | 里程/速度基准 | 实时速度 v(t) |
| 5 | 工业控制计算机 | 1 台 | 融合计算 | EKF 状态估计、TQI 统计 |

注：单点测距传感器用于直接测量轨面垂向距离（高低检测），3D 线激光传感器用于钢轨廓形检测（波磨/磨耗/横移）。本系统不使用磁力计，不依赖地磁定北，不受铁轨地磁干扰影响。

### 2.3 物理模型与符号约定

| 符号 | 含义 | 单位 |
|------|------|------|
| z = 0 | 基准参考面（海平面） | — |
| h_track(k) | 轨道表面高程偏差（相对长期均值） | mm |
| h_veh(k) | 车体跳动位移（向上为正） | mm |
| H | 激光器安装高度（相对车体基准点，静态校准值） | mm |
| H | 单点测距传感器安装高度（相对车体基准点，静态校准值，约30mm）| mm |
| d_测(k) | 单点测距传感器沿轴向测量距离（传感器到轨面垂距）| mm |
| T(k) | 传感器环境温度（用于温漂补偿） | °C |
| θ_pitch(k) | 车体俯仰角 | rad |
| θ_roll(k) | 车体横滚角 | rad |

**测距方程（经验证，正确）：**

```
几何关系：
d_测(k) = 激光器垂向位置 - 轨道表面垂向位置
        = (H + h_veh(k)) - h_track(k)

物理验证：
- 车体上跳 h_veh ↑ → 激光器远离轨面 → d_测 ↑ ✅
- 轨道抬高 h_track ↑ → 轨面更近 → d_测 ↓ ✅

整理：
h_track(k) = H + h_veh(k) - d_测(k)          （式1）

⚠️ 注意符号：不是"高程+跳动"，而是"跳动-高程"
```

### 2.4 车体跳动与轨道不平顺的频率重叠分析

**v = 1m/s 时，波长-频率换算 λ = v/f：**

| 波长区间 | 频率范围 | 主要成分 | EKF 可分离性 |
|---------|---------|---------|-------------|
| λ ≥ 10m | f < 0.1Hz | h_track 为主（缓变） | ✅ 良好 |
| 1m ≤ λ < 10m | 0.1~1Hz | h_track 和 h_veh 量级相近 | ⚠️ 中等（频率重叠） |
| λ < 1m | f > 1Hz | h_track → 0，h_veh 主导 | ✅ 良好（可忽略 h_track） |

**关键结论：**
在 1m ≤ λ < 10m 波段，h_track 和 h_veh 频率高度重叠，EKF 协方差会偏大，该波段测量精度相对较低。缓解策略见 2.9 节。

### 2.5 状态空间模型

**状态向量（修正：去除 v_track 冗余状态）：**

```
X(k) = [h_track(k), h_veh(k), v_veh(k)]ᵀ
     = [轨道高程偏差, 车体跳动位移, 车体跳动速度]ᵀ
```

**注：v_track 从状态向量中移除。理由：**
- v_track 无实际物理用途（既不参与预测也不参与输出）
- 轨道沉降不是匀速过程，"速度"概念含糊
- h_track 改用随机游走模型（Random Walk）更合理

**状态转移矩阵（修正：以里程增量 Δs 为步长）：**

```
X(k+1) = F · X(k) + W(k)

     1    0    0
F =  0    1   Δs    ← h_veh(k+1) = h_veh(k) + v_veh(k)·Δs
     0    0    1     ← v_veh(k+1) = v_veh(k)

其中：Δs = v(k)·Δt （轮速编码器积分，单位 km）
W(k) ~ N(0, Q)
```

**观测方程：**

```
Z(k) = d_测(k) = H + h_veh(k) - h_track(k) + V(k)
     = C·X(k) + H + V(k)

C = [-1, 1, 0]
V(k) ~ N(0, R)
```

**能观性分析：**

```
能观性矩阵 O = [Cᵀ; Cᵀ·Fᵀ; Cᵀ·(Fᵀ)²]ᵀ

对于 3 状态系统：
只要 Δs ≠ 0（车在运动），系统局部能观 ✅

⚠️ 能观性≠分离精度：
   在 1m ≤ λ < 10m 波段，两状态频谱重叠
   EKF 协方差偏大，估计不确定性增加
   这是本方案已知的精度局限性
```

### 2.6 俯仰角校正（修正：余弦公式改正）

**正确公式（⚠️ 修正 `/cos` → `·cos`）：**

```
激光测的是沿轴向距离 d_测
当存在俯仰角 θ 时，竖直方向分量：

d_校正 = d_测 · cos(θ)

物理验证：
θ = 0°  → cos(0) = 1    → d_校正 = d_测（无倾斜，无影响）✅
θ = 10° → cos(10°)≈0.985 → d_校正 < d_测（倾斜后竖直距离缩短）✅

⚠️ 之前错误写成 d_测 / cos(θ)，方向反了！
```

**横滚角对高程测量的影响（补充）：**

```
横滚角 θ_roll 主要影响左右激光的横向坐标 y，不直接影响 d_测
但会影响廓形横截面形状：

横滚时，同一横断面的左右测量点有横向偏移
→ 轨头横断面的"最高点"位置 y_peak 会有横向偏移
→ 若直接取 max(z) 而不拟合廓形，会提取错误的 z 值

处理方法：使用抛物线/高斯拟合求峰值，而非直接取 max(z)（见 2.12）
```

**左右激光分别校正：**

```
θ_i(k) = θ_pitch(k) + L_i · pitch_rate(k) / v(k)
d_i_校正(k) = d_i_测(k) · cos(θ_i(k))

⚠️ 倾角传感器在车体中部，不是激光器位置
   L_i 为激光 i 相对俯仰旋转中心的纵向距离
   pitch_rate 由陀螺仪角速率数据计算
```

### 2.7 温度漂移补偿

**温漂来源分析（来自传感器 datasheet）：**

| 传感器 | 温漂参数 | 等效垂向误差（积分10s后）|
|--------|---------|------------------------|
| 加速度计 | ±0.09 mg/°C | ±0.44 mm/°C |
| 陀螺仪 | ±0.005~0.015 °/s/°C | ±0.17 mm/°C（取典型值0.01°/s）|

**问题估算：**
```
运行2小时，环境温度变化20°C：
→ 累积加速度温漂误差 ≈ ±8.8 mm（不可忽略！）

但注意：这是积分累积误差，实际系统不会积分那么久
→ EKF 的过程噪声 Q 会抑制无限累积
→ 温漂主要造成偏置误差，而非无限增长
```

**补偿方案：**

```
方案A（实时补偿，推荐）：
T_补偿(k) = α_a · (T(k) - T_ref) + β_a
  — α_a, β_a 由温度标定实验确定
  — 每帧同时修正加速度计和陀螺仪输出

方案B（定期零点校准）：
— 检测开始前，停车10min，取温度和零点稳定数据
— 运行中每30min短停5s，重新校准零点

本方案采用方案A+方案B结合
```

### 2.8 EKF 算法

**参数初始化（修正：去除 v_track，对应3状态）：**

```
X̂(0) = [d(0)-H, 0, 0]ᵀ
      — 初始高程 = 首帧激光读数减去安装高度
      — 初始跳动和跳动速度设为 0

⚠️ 改进建议：正式检测前以 <0.1m/s 行驶 20m，取前 10m 均值作为初始校准值
   避免首帧恰好在轨道接头处引入初始偏差

P(0) = diag([0.5², 0.5², 0.1²])  mm²

Q 矩阵（过程噪声协方差，3状态）：

Q = diag([q_h, q_h_veh, q_v_veh])

其中（⚠️ 修正量纲和概念区分）：

q_h ≈ 1×10⁻⁶ mm²/s
  — 轨道高程随机游走强度（纯位置随机游走谱密度）
  — 量纲：mm²/s（是位置，不是加速度）
  — 来源：纯数学建模，无需传感器 Allan 方差

q_h_veh ≈ 0.01 mm²/s
  — 车体跳动位移驱动噪声谱密度
  — ⚠️ 注意单位是 mm²/s，不是 mm²/s³！
  — 这是整车振动环境估算值（包含悬挂+轨道激振）
  — 远大于传感器裸机 Allan 方差地板
  — 两者区分：传感器 Allan 方差是"仪器精度下限"，q_h_veh 是"实际使用环境的等效噪声水平"

q_v_veh ≈ 0.001 mm²/s²
  — 车体跳动速度过程噪声谱密度

⚠️ Allan 方差和 Q 矩阵的关系（补充说明）：
  传感器 datasheet 中的"零偏不稳定性"（BI）是 σ(τ) 在某一 τ 下的等效读数噪声
  不等于 Q 矩阵直接取值
  Q 矩阵应通过：
  ① 静态 Allan 方差实验 → 确定传感器噪声结构
  ② 整车振动测试 → 确定实际使用环境的 q_h_veh 上限
  最终 Q 值取两者中的实际主导项

R（观测噪声）：
R = σ_laser² ≈ (0.05mm)² = 0.0025 mm²
```

**EKF 预测步：**

```
X̂⁻(k) = F(k-1) · X̂(k-1)
P⁻(k)  = F(k-1) · P(k-1) · F(k-1)ᵀ + Q
```

**EKF 更新步：**

```
ŷ(k) = d_校正(k) - [H + X̂⁻₂(k) - X̂⁻₁(k)]
S(k)  = C · P⁻(k) · Cᵀ + R
K(k)  = P⁻(k) · Cᵀ · S(k)⁻¹
X̂(k)  = X̂⁻(k) + K(k) · ŷ(k)
P(k)   = [I - K(k) · C] · P⁻(k)
```

**输出：**

```
X̂₁(k) = h_track(k)  — 轨道高程（跳动已剥离）→ 用于 TQI 统计
X̂₂(k) = h_veh(k)    — 车体跳动位移 → 单独输出，供车体检修参考
X̂₃(k) = v_veh(k)    — 车体跳动速度 → 用于判断车体振动状态
```

### 2.9 中波区间精度限制与缓解策略

**已知局限：1m ≤ λ < 10m 波段，EKF 分离能力下降**

**缓解策略：**

```
策略1：利用 h_track 和 h_veh 不同的统计特性
  — h_track：空间相关性高（相邻采样点高度相关）
  — h_veh：时间相关性低（振动是局部事件）
  — 在 EKF 更新后增加空间平滑约束：
    h_track_平滑(k) = γ·h_track(k) + (1-γ)·h_track_邻居(k)
    其中 γ 由实验确定（典型值 0.7~0.9）

策略2：利用波长先验
  — 已知铁路轨道在 12.5m 和 25m 处有特征波长（钢轨轧制残余波）
  — 在这些特征波长处增加先验约束，辅助 EKF 分离

⚠️ 以上策略为可选增强，不影响基础方案正确性
   论文中应如实说明中波区间是当前方案的已知局限性
```

### 2.11 车体高频跳动抑制

**自适应陷波滤波：**

```python
def adaptive_notch(signal, f_veh, fs, Q=30):
    w0 = 2π · f_veh / fs
    α = sin(w0) / (2Q)
    b = [1, -2cos(w0), 1] / (1 + α)
    a = [1, -2cos(w0), 1 - α]
    return lfilter(b, a, signal)
```

**滑动窗口多项式趋势项去除：**

```
窗口：L = 200采样点 @2000Hz, 1m/s → 0.1m 空间窗口
拟合：y_fit(k) = a₀ + a₁·k + a₂·k²
趋势项 = y_fit    — 车体缓慢起伏（弃用）
波动项 = y - y_fit — 真正的轨道不平顺（保留）
```

### 2.12 3D 激光廓形高程提取（修正：峰值提取改用拟合）

**廓形点云处理：**

```
每帧廓形点集：P = {(x_i, y_i, z_i), i = 1...N}

步骤1：离群点去除（统计滤波，>2σ 删除）
步骤2：光顺滤波（中值滤波，窗口3点）
步骤3：轨头边界检测（y 方向梯度最大点 → 左/右边界）
```

**步骤4：抛物线拟合求峰值（⚠️ 修正：不用 max()，改用拟合）：**

```
⚠️ 直接取 max(z) 的问题：
  当轨头表面有缺陷（凹陷、掉块、波磨）时
  最大值点可能落在缺陷旁边，得到错误的 z_peak

正确方法：抛物线拟合

在 z 方向最大值点附近取前后各3点（共7点）：
(y₁,z₁), (y₂,z₂), ..., (y₇,z₇)，y₂ 为峰值附近

拟合抛物线：
z(y) = a·y² + b·y + c

最小二乘求解系数 a, b, c：
y_peak = -b/(2a)      ← 亚像素峰值位置
z_peak = z(y_peak)   ← 亚像素峰值高程

精度：从 ±1 pixel 提升到 ±0.1 pixel ✅

⚠️ 当轨头缺陷较严重（缺损 > 30% 轨头宽度）时
   抛物线拟合也会失效
   需增加缺损检测逻辑，标记该帧数据为"缺损区域，不参与统计"
```

### 2.13 完整信号处理流程

```
原始激光 d_L(k), d_R(k)  [2000Hz]
        │
        ├─→ 倾角传感器 → 分别算 θ_L, θ_R
        │   → d_corr = d_测 · cos(θ)    （⚠️ 修正：乘，不是除）
        │
        ├─→ 加速度计 → Allan 方差标定Q + 温度监测触发温漂补偿
        │
        └─→ 轮速编码器 v(k) → Δs = v·Δt → F矩阵
        │
        ▼
   EKF状态估计（同时估计 h_track 和 h_veh）
        │
        ├──→ 自适应陷波滤波（剔除 f_veh）
        └──→ 滑动窗口多项式（去除趋势项）
        │
        ▼
   带通滤波 → 高通(λ<10m)→σ_短波，低通(λ≥10m)→σ_长波
        │
        ▼
   200m 单元 TQI 统计 → TB/T 3355-2023 七项制
```

### 2.14 TQI 计算（补充：明确左右高低独立）

**TB/T 3355-2023 七项制：**

```
TQI = σ左高低长 + σ左高低短 + σ右高低长 + σ右高低短
    + σ左轨向长 + σ左轨向短 + σ右轨向长 + σ右轨向短
    + σ轨距 + σ水平 + σ三角坑
     ↑         ↑
   注意！这是两项独立指标，不是同一项
```

**⚠️ 重要澄清：**
- 左高低和右高低是独立计算的两项（对应左轨和右轨各自的检测数据）
- 不是"左右轨高差"，而是"左轨自身高程序列的标准差 + 右轨自身高程序列的标准差"
- 本系统的左右两路 3D 激光分别处理，可分别输出左/右轨 TQI 分项
