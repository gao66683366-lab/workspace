# 第8章 智能缺陷识别与报告平台

## 8.1 平台定位

智能缺陷识别与报告平台是本系统的数据汇聚与价值变现核心，以大数据累积为根基，承载检测数据的分析、可视化与决策支持功能，为轨道交通运维提供智能化工具。

## 8.2 数据来源

| 数据来源 | 数据类型 | 说明 |
|---|---|---|
| 本机实时采集 | 视频流、缺陷图片、传感器数据 | 现场边检边采 |
| 历史检测记录 | 报告、缺陷数据库 | 历史积累，支持趋势分析 |
| 外部系统接入 | MRO/EAM数据接口 | 车辆修程、维修记录 |
| GIS地理信息 | 段场/股道/位置映射 | 定位数据关联 |

## 8.3 智能缺陷识别流程

```
[原始图像帧]
    ↓
[图像预处理] → 去噪声、低照度增强、去油污反光
    ↓
[AI缺陷检测] → 目标检测（YOLO）+ 缺陷分割（Mask）
    ↓
[缺陷分级] → 严重/一般/轻微（置信度阈值+形态学后处理）
    ↓
[结果输出] → 缺陷框+类别+置信度+像素级Mask
    ↓
[结果反馈] → 实时推送到手持终端 + 本地存储 + 报告归档
```

**识别处理指标：**

| 指标 | 目标值 |
|---|---|
| 单帧处理时间 | <200ms（iGPU + OpenVINO INT8推理） |
| 支持缺陷类别 | >=20类 |
| 识别准确率 | >=95% |
| 误报率 | <=5% |

## 8.4 报告自动生成系统

### 8.4.1 报告模板体系

| 模板类型 | 适用场景 |
|---|---|
| 铁路通用模板 | 地铁、动车、客车、货车 |
| 大机段专用模板 | 捣固车、稳定车、清筛车等 |

### 8.4.2 报告内容结构

| 章节 | 内容 |
|---|---|
| 报告封面 | 车辆信息（车型、车号）、检测时间、检测人员 |
| 检测概况 | 检测总里程、检测时长、采集图片数、缺陷总数 |
| 缺陷明细 | 缺陷列表（含图片、位置、类型、等级、处理建议） |
| 统计图表 | 缺陷类型分布图、部位分布图、等级分布饼图 |
| 附件 | 原始视频片段（可选）、AI推理日志 |
| 审核签章 | 检测员签章、审核员签章、日期 |

### 8.4.3 报告输出格式

| 格式 | 说明 |
|---|---|
| PDF | 正式存档格式，支持电子签章 |
| Excel | 数据表格格式，便于汇总统计 |
| 本地保存 | 自动归档至本地存储 |
| 打印 | 支持A4/A3纸打印 |
| 云端上传 | 自动推送至MRO/运维平台 |

## 8.5 GIS可视化（地理信息系统）

| 功能 | 说明 |
|---|---|
| 股道地图 | 显示段场股道平面图，标注机器人行走轨迹 |
| 缺陷位置标记 | 在地图上标注缺陷位置，可点击查看详情 |
| 热力图 | 按区域显示缺陷密度分布 |
| 行走回放 | 按时间轴回放历史检测轨迹 |
| 统计汇总 | 按股道/车型/时间段汇总统计 |

## 8.6 预测性维护支持

| 功能 | 说明 |
|---|---|
| 趋势分析 | 同一车辆的历次检测数据对比，追踪缺陷变化趋势 |
| 劣化预测 | 基于历史数据，预测部件剩余寿命 |
| 检修建议 | 根据缺陷趋势自动生成优先检修建议 |
| 报表导出 | 支持月度/季度/年度报表导出 |

## 8.7 系统对接接口

| 对接系统 | 接口方式 | 数据内容 |
|---|---|---|
| MRO系统 | RESTful API / WebService | 检测报告、缺陷数据 |
| EAM系统 | RESTful API / 数据库直连 | 车辆资产数据、维修记录 |
| 工务检修平台 | RESTful API / FTP | 检测数据文件包 |
| 铁路95586客服 | RESTful API（可选） | 异常缺陷自动上报 |
