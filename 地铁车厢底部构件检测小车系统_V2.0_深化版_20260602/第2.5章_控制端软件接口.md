# 第2.5章 控制端软件接口

> **郑重声明：本章接口规范经严格设计，适用于WiFi无线链路（延迟10~50ms、偶发丢包、连接不稳定）场景。所有协议设计均考虑无线通信的边界条件，不可省略任何可靠性机制。**

## 2.5.0 设计背景与约束

### 2.5.0.1 无线链路特性

| 特性 | 参数 | 对协议的影响 |
|---|---|---|
| **典型延迟** | 10~50ms（端到端）| 控制指令需等待应答超时设计 |
| **最大延迟** | 200ms（拥堵时）| 超时阈值不得低于500ms |
| **丢包率** | 0.1%~2%（正常）| 必须有重传机制 |
| **抖动** | ±20ms | 不得依赖精确时序 |
| **断线风险** | 可能随时断开 | 必须有断线检测和自动重连 |
| **带宽** | 300~600Mbps（有效）| 视频码率可动态调节 |

### 2.5.0.2 协议设计原则

1. **所有指令必须应答**：无应答即重发，防止石沉大海
2. **幂等性设计**：重复指令不产生副作用
3. **序列号防重放**：防止网络延迟导致指令重复执行
4. **断线自动重连**：不需要人工干预
5. **数据完整性校验**：CRC16/CRC32保护

## 2.5.2 通信矩阵

| 通道 | 协议 | 端口 | 方向 | 用途 | 可靠性 |
|---|---|---|---|---|---|
| **控制指令** | TCP | 5000 | 双向 | 运动/云台/光源/采集控制指令 | **可靠**（应答确认）|
| **状态数据** | TCP | 5001 | 主控板→控制端 | 系统状态、错误码、ACK | **可靠** |
| **传感器数据** | UDP | 5002 | 主控板→控制端 | IMU/测距/里程/BMS/激光雷达 | **不可靠**（允许丢包）|
| **视频流** | RTSP/RTP | 8554 | 主控板→控制端 | 前视/后视实时视频 | **不可靠**（UDP）|
| **发现服务** | UDP | 5004 | 主控板→控制端 | 设备发现/广播IP | **不可靠** |

## 2.5.3 控制指令接口（TCP:5000）

### 2.5.3.1 帧格式定义（统一）

| 字段 | 长度 | 说明 |
|---|---|---|
| Header | 2字节 | 固定 0xAA 0x55，帧起始标志 |
| Sequence | 2字节 | **序列号**（0~65535，递增），用于应答匹配和重发去重 |
| Length | 2字节 | 后续字段总长度（CMD + Payload） |
| CMD | 1字节 | 主命令字 |
| Payload | N字节 | 参数数据，格式视CMD而定 |
| Checksum | 2字节 | **CRC16**（覆盖Header~Payload，不含Checksum自身）|

### 2.5.3.2 应答机制（必须实现）

**每条指令必须等待应答**，超时重发：

| 项目 | 参数 | 说明 |
|---|---|---|
| **超时时间** | 500ms | 500ms内未收到应答则重发 |
| **最大重发次数** | 3次 | 3次重发均无应答则判定连接断开 |
| **重发间隔** | 200ms | 重发前等待200ms |
| **序列号作用** | 应答匹配 | 主控板返回相同Sequence的ACK/NACK |

**应答状态码：**

| Status | 含义 |
|---|---|
| 0x00 | ACK - 执行成功 |
| 0x01 | NACK - 命令不支持 |
| 0x02 | NACK - 参数错误 |
| 0x03 | NACK - 序列号重复（非首发送）|
| 0x04 | NACK - 系统忙，稍后重试 |
| 0xFF | NACK - 未知错误 |

### 2.5.3.3 命令字定义

#### 0x01 运动控制

| subcmd | 功能 | Payload | 说明 |
|---|---|---|---|
| 0x01 | 设置速度 | float32 speed (m/s) | 范围0.0~0.5 |
| 0x02 | 前进 | 无 | 以当前设定速度前进 |
| 0x03 | 后退 | 无 | 以当前设定速度后退 |
| 0x04 | 停止 | 无 | 平滑减速停止 |
| 0x05 | **急停** | 无 | 立即切断电机输出，最快响应 |

#### 0x02 云台控制

| subcmd | 功能 | Payload | 说明 |
|---|---|---|---|
| 0x01 | 角度控制 | uint8 cam + int16 pan + int16 tilt | cam:0前/1后 |
| 0x02 | 云台复位 | uint8 cam | 回到中立位(pan=0, tilt=0) |
| 0x03 | 预设位保存 | uint8 cam + uint8 preset_id | 保存当前角度 |
| 0x04 | 预设位调用 | uint8 cam + uint8 preset_id | 调到预设角度 |

#### 0x03 光源与照明控制

| subcmd | 功能 | Payload | 说明 |
|---|---|---|---|
| 0x01 | 设置主光源亮度 | uint8 brightness | 0~100 (%) |
| 0x02 | 前照明灯开关 | uint8 onoff | 0=关, 1=开 |
| 0x03 | 后照明灯开关 | uint8 onoff | 0=关, 1=开 |

#### 0x04 采集控制

| subcmd | 功能 | Payload | 说明 |
|---|---|---|---|
| 0x01 | 开始采集 | uint8 mode | 0=自动模式, 1=手动模式 |
| 0x02 | 停止采集 | 无 | 停止本次采集 |
| 0x03 | 保存数据 | 无 | 强制保存当前数据到本地 |

#### 0x05 系统控制

| subcmd | 功能 | Payload | 说明 |
|---|---|---|---|
| 0x01 | 系统自检 | 无 | 请求主控板执行自检并返回结果 |
| 0x02 | 零点标定 | uint8 type | 0=里程计, 1=姿态传感器 |
| 0x03 | 时间同步 | uint64 timestamp | Unix时间戳（毫秒）|
| 0x04 | 请求系统状态 | 无 | 立即获取当前系统状态 |
| 0x05 | 进入休眠 | 无 | 主控板进入低功耗待机 |
| 0x06 | 唤醒 | 无 | 从休眠唤醒 |

#### 0xFF 心跳与存活确认

| subcmd | 功能 | Payload | 说明 |
|---|---|---|---|
| 0x01 | **心跳** | uint32 client_time（可选）| 客户端每2秒发送一次，丢失3次则判定断开 |
| 0x02 | **查询连接状态** | 无 | 查询主控板当前连接状态 |

## 2.5.4 状态数据接口（TCP:5001）

### 2.5.4.1 状态类型定义

| Type | 说明 | 触发条件 |
|---|---|---|
| 0x01 | **ACK/NACK** | 收到控制指令后的应答 |
| 0x02 | **系统状态** | 周期1Hz推送 or 应答查询指令 |
| 0x03 | **自检结果** | 应答系统自检指令 |
| 0x04 | **报警事件** | 异常发生时立即推送 |
| 0x05 | **连接状态** | 应答连接状态查询 |

### 2.5.4.2 系统状态帧（Type=0x02）

| 字段 | 类型 | 说明 |
|---|---|---|
| mode | uint8 | 0=待机, 1=自动, 2=手动, 3=应急 |
| error_code | uint8 | 0=正常, >0=错误码 |
| cpu_temp | int16 | CPU温度，0.1℃（例：355=35.5℃）|
| memory_usage | uint16 | 内存使用率，0.1%（例：502=50.2%）|
| board_temp | int16 | 板级温度，0.1℃ |
| wifi_rssi | uint8 | WiFi信号强度，dBm（例：-65）|
| wifi_quality | uint8 | WiFi质量 0~100% |
| uptime_seconds | uint32 | 运行时长，秒 |

### 2.5.4.3 报警事件帧（Type=0x04）

| 字段 | 类型 | 说明 |
|---|---|---|
| alarm_level | uint8 | 1=Info, 2=Warning, 3=Error, 4=Critical |
| alarm_code | uint8 | 报警编码 |
| alarm_data | uint16 | 附加数据 |
| timestamp | uint32 | Unix时间戳，秒 |
| description | char[32] | 报警描述文本（UTF-8）|

## 2.5.5 传感器数据接口（UDP:5002）

> **重要：UDP通道为不可靠通道，数据可能丢失或乱序。所有数据仅供显示参考，不应用于控制决策。**

### 2.5.5.1 UDP帧格式

| 字段 | 长度 | 说明 |
|---|---|---|
| Header | 1字节 | 固定 0xAA |
| Timestamp | 4字节 | Unix时间戳（毫秒）|
| Sequence | 2字节 | 序列号（可能不连续，因UDP丢包）|
| Payload | N字节 | 数据内容 |
| Checksum | 4字节 | CRC32校验 |

### 2.5.5.2 传感器数据类型

| Sensor_Type | 数据类型 | 频率 | 说明 |
|---|---|---|---|
| 0x01 | **IMU数据** | 50Hz | 姿态传感器 |
| 0x02 | **测距数据** | 20Hz | 前后测距 |
| 0x03 | **里程数据** | 50Hz | 行走距离 |
| 0x04 | **BMS数据** | 1Hz | 电池管理 |
| 0x05 | **激光雷达** | 10Hz | 点云数据 |

### 2.5.5.3 IMU数据（Sensor_Type=0x01, 50Hz）

| 字段 | 类型 | 说明 |
|---|---|---|
| roll | float | 横滚角，度，范围-180~+180 |
| pitch | float | 俯仰角，度，范围-90~+90 |
| yaw | float | 航向角，度，范围0~360 |
| wx, wy, wz | float | 角速度X/Y/Z，deg/s |
| ax, ay, az | float | 加速度X/Y/Z，g |

### 2.5.5.4 测距数据（Sensor_Type=0x02, 20Hz）

| 字段 | 类型 | 说明 |
|---|---|---|
| front | float | 前方距离，mm |
| rear | float | 后方距离，mm |

### 2.5.5.5 里程数据（Sensor_Type=0x03, 50Hz）

| 字段 | 类型 | 说明 |
|---|---|---|
| distance | float | 累计里程，mm |
| speed | float | 当前速度，m/s |

### 2.5.5.6 BMS数据（Sensor_Type=0x04, 1Hz）

| 字段 | 类型 | 说明 |
|---|---|---|
| voltage | float | 总电压，V |
| current | float | 电流，A（正=充电，负=放电）|
| soc | uint8 | 荷电状态，% |
| temp_max | int8 | 最高温度，℃ |
| temp_min | int8 | 最低温度，℃ |
| cycle_count | uint16 | 循环次数 |
| status | uint8 | 状态位 |

**BMS状态位定义：**

| Bit | 定义 |
|---|---|
| Bit0 | 0=正常 / 1=告警 |
| Bit1 | 0=未过温 / 1=过温 |
| Bit2 | 0=未欠压 / 1=欠压 |
| Bit3 | 0=未过流 / 1=过流 |
| Bit4 | 0=未短路 / 1=短路保护 |
| Bit5 | 0=充满 / 1=未充满 |

## 2.5.6 视频流接口（RTSP/RTP）

### 2.5.6.2 视频参数

| 项目 | 参数 |

### 2.5.6.3 码率自适应策略

| RSSI范围 | 视频质量 |
|---|---|

## 2.5.7 设备发现接口（UDP:5004）

机器人上电后通过UDP广播设备信息，控制端通过发送查询帧获取设备IP。

**控制端查询帧（广播255.255.255.255:5004）：**

| 字段 | 值 | 说明 |
|---|---|---|
| Byte[0] | 0xFF | 发现请求标识 |
| Byte[1] | 0x01 | 版本号 |

**主控板响应帧（单播回原端口）：**

| 字段 | 类型 | 说明 |
|---|---|---|
| Byte[0] | 0xFE | 发现响应标识 |
| Byte[1] | 0x01 | 版本号 |
| Byte[2-5] | uint32 | 机器人IP（little-endian）|
| Byte[6-9] | uint32 | 子网掩码 |
| Byte[10-13] | uint32 | 网关 |
| Byte[14-17] | uint32 | DNS |
| Byte[18-25] | char[8] | 设备ID（如ROBOT001）|
| Byte[26-29] | uint32 | 固件版本 |
| Byte[30-33] | uint32 | 能力掩码 |

## 2.5.9 章节小结

本章定义了Windows触控屏控制软件与机器人主控板之间的完整软件接口规范，针对WiFi无线链路特性做了充分设计：

- **通信矩阵**：TCP控制(5000)/TCP状态(5001)/UDP传感器(5002)/RTSP视频(8554)/设备发现(5004)
- **帧格式**：统一格式（Header+Sequence+Length+CMD+Payload+Checksum），CRC16校验
- **应答机制**：500ms超时，3次重发，序列号匹配，防重放
- **命令体系**：0x01运动/0x02云台/0x03光源/0x04采集/0x05系统/0xFF心跳
- **传感器数据**：IMU(50Hz)/测距(20Hz)/里程(50Hz)/BMS(1Hz)/激光雷达(10Hz)
- **视频流**：RTSP双通道，低延迟模式，码率自适应
- **设备发现**：UDP广播查询，自动获取机器人IP
- **SDK封装**：C#完整API + Python示例代码

以上接口规范可用于Windows控制软件的开发和移植。
